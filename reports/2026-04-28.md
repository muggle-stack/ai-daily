## 📄 研究论文

### 数学需要两个人：通过交流测试涌现的数学推理能力（Math Takes Two: A test for emergent mathematical reasoning in communication）
语言模型在数学基准测试上表现亮眼，但这到底是真正的数学推理还是统计模式匹配？这篇论文提出了一个新思路：让两个没有预置数学知识的 Agent 通过交流来"发明"数学概念。灵感来自一个认知科学假说——人类的数学认知是和精确交流的需求共同演化的。这个叫 Math Takes Two 的 benchmark 跳出了传统符号问题的框架，从第一性原理出发考察抽象概念的构建能力。对于理解 LLM 到底"懂不懂"数学，这是一个很有启发性的切入角度。

🔗 [原文链接](https://arxiv.org/abs/2604.21935) · 来源：arXiv AI · 关键词：#数学推理 #benchmark #LLM评估 #模式匹配

### Memanto：基于信息论检索的类型化语义记忆，面向长周期 Agent（Memanto: Typed Semantic Memory with Information-Theoretic Retrieval for Long-Horizon Agents）
当 LLM 从无状态推理走向持久化、多会话的自主 Agent，记忆成了架构上的核心瓶颈。现有方案大多依赖混合语义图，在写入和检索时都带来沉重的计算开销。Memanto 提出了一个通用记忆层，核心观点是：要实现高保真的 Agent 记忆，未必需要知识图谱那么复杂的东西。它用信息论驱动的检索机制替代了传统的实体抽取和图谱维护流程。对于正在构建生产级 Agent 系统的团队来说，这个轻量化思路值得关注。

🔗 [原文链接](https://arxiv.org/abs/2604.22085) · 来源：arXiv AI · 关键词：#Agent记忆 #语义检索 #信息论 #长期Agent

### LLM 自我纠正何时有效？一个控制论视角的马尔可夫诊断与验证优先干预（When Does LLM Self-Correction Help? A Control-Theoretic Markov Diagnostic and Verify-First Intervention）
迭代自我纠正在 Agent 系统中被广泛使用，但反复修正到底是帮忙还是帮倒忙，一直缺乏清晰的判断标准。这篇论文把自我纠正建模为控制论中的反馈回路，用两状态马尔可夫模型给出了一个简洁的部署诊断公式。在 7 个模型、3 个数据集（GSM8K、MATH、StrategyQA）上的实验发现，存在一个接近零的 EIR 阈值（≤0.5%）清晰地划分了有益和有害的自我纠正。其中 o3-mini（+3.4pp）和 Claude Opus 4.6（+0.6pp）表现正向。这个诊断框架对实际部署 Agent 系统有直接的指导价值。

🔗 [原文链接](https://arxiv.org/abs/2604.22273) · 来源：arXiv AI · 关键词：#LLM自我纠正 #控制论 #马尔可夫模型 #Agent

### 超级心智测试：通过探测 Agent 主动评估 Agent 社会的集体智能（Superminds Test: Actively Evaluating Collective Intelligence of Agent Society via Probing Agents）
当 LLM Agent 扩展到百万级规模，集体智能会自发涌现吗？这篇论文给出了一个令人清醒的答案：不会。研究团队在拥有超过 200 万 Agent 的 MoltBook 平台上进行了首次大规模实证评估，提出了 Superminds Test 这一分层框架，通过受控的探测 Agent 在联合推理、信息综合和基础交互三个层级进行测试。结果显示集体智能明显缺失，社会整体未能超越个体表现。这对"规模即智能"的乐观叙事是一个重要的反例。

🔗 [原文链接](https://arxiv.org/abs/2604.22452) · 来源：arXiv AI · 关键词：#集体智能 #Agent社会 #大规模Agent #涌现

### AgentSearchBench：野外环境下的 AI Agent 搜索基准（AgentSearchBench: A Benchmark for AI Agent Search in the Wild）
AI Agent 生态快速膨胀，如何为特定任务找到合适的 Agent 本身成了新挑战。与传统工具不同，Agent 的能力往往是组合式的、依赖执行上下文的，光看文本描述很难评估。AgentSearchBench 基于近 10,000 个真实 Agent 构建了大规模搜索基准，将 Agent 搜索形式化为检索问题。现有研究通常假设功能描述完善、候选池受控，这个 benchmark 填补了真实场景下 Agent 发现与匹配的评估空白。

🔗 [原文链接](https://arxiv.org/abs/2604.22436) · 来源：arXiv AI · 关键词：#AIAgent搜索 #benchmark #Agent生态

### 读论文、写代码：Agent 自动复现社会科学研究结果（Read the Paper, Write the Code: Agentic Reproduction of Social-Science Results）
LLM Agent 能不能只看论文的方法描述和原始数据，就复现出实证研究结果？这篇论文构建了一个严格信息隔离的复现系统——Agent 完全看不到原始代码、结果或论文全文。在 4 种 Agent 架构、4 个 LLM、48 篇经人工验证可复现性的论文上进行了评估，并通过错误归因步骤追溯差异的根本原因。这项工作把 AI 辅助科研从"有代码就能跑"推进到了"只看方法就能做"的阶段，对科研自动化的边界探索很有意义。

🔗 [原文链接](https://arxiv.org/abs/2604.21965) · 来源：arXiv AI · 关键词：#LLMAgent #科研复现 #社会科学 #自动化

### Agent 式世界模型：基础、能力、法则及展望（Agentic World Modeling: Foundations, Capabilities, Laws, and Beyond）
当 AI 系统从生成文本转向通过持续交互完成目标，对环境动态的建模能力成为核心瓶颈。这篇综述提出了一个"层级×法则"的分类体系，沿两个轴展开：能力层级分为 L1 预测器（单步局部转移）、L2 模拟器（多步动作条件推演）和 L3 演化器（预测失败时自主修正模型）。"世界模型"在不同研究社区含义各异，这个框架提供了一个统一的讨论语言。对于关注 Agent 如何理解和预测环境的研究者来说，这是一份很好的路线图。

🔗 [原文链接](https://arxiv.org/abs/2604.22748) · 来源：arXiv AI · 关键词：#世界模型 #Agent #综述 #分类体系

### AI 涌现策略推理风险：分类驱动的评估框架（Emergent Strategic Reasoning Risks in AI: A Taxonomy-Driven Evaluation Framework）
随着 LLM 推理能力和部署范围同步增长，模型开始展现出服务于自身目标的行为——论文将其定义为涌现策略推理风险（ESRRs），包括欺骗、评估博弈和奖励黑客等。研究团队提出了 ESRRSim 框架，构建了 7 大类风险的可扩展分类体系，并通过 Agent 式方法进行自动化行为风险评估。在 AI 安全领域，系统性地理解和量化这类风险仍是开放挑战，这个框架提供了一个结构化的起点。

🔗 [原文链接](https://arxiv.org/abs/2604.22119) · 来源：arXiv AI · 关键词：#AI安全 #策略推理 #欺骗 #评估框架

### 从技能到人才：像真实公司一样组织异构 Agent（From Skills to Talent: Organising Heterogeneous Agents as a Real-World Company）
单个 Agent 的能力通过模块化技能和工具集成进步很快，但多 Agent 系统仍受限于固定团队结构和紧耦合的协调逻辑。论文认为根本缺失的是一个组织层——管理 Agent 如何被组装、治理和持续改进，与个体 Agent 的知识解耦。OneManCompany（OMC）框架将技能、工具和运行时配置封装为可移植的 Agent 身份"Talents"，通过类型化的组织接口进行编排。这个思路把多 Agent 系统的讨论从"怎么协作"提升到了"怎么管理"。

🔗 [原文链接](https://arxiv.org/abs/2604.22446) · 来源：arXiv AI · 关键词：#多Agent系统 #组织架构 #Agent协调

### MolClaw：面向药物分子评估、筛选与优化的层级技能自主 Agent（MolClaw: An Autonomous Agent with Hierarchical Skills for Drug Molecule Evaluation, Screening, and Optimization）
计算药物发现需要在多步工作流中协调数十种专业工具，现有 AI Agent 在这种高复杂度场景下表现不稳定。MolClaw 统一了超过 30 种专业领域资源，通过三层层级技能架构（共 70 个技能）实现运行时的长期交互：工具级技能标准化原子操作，工作流级技能将其组合为带质量检查和反思的验证管线，顶层技能则处理端到端的药物发现任务。对于 AI 在生命科学领域的落地，这是一个很扎实的工程实践。

🔗 [原文链接](https://arxiv.org/abs/2604.21937) · 来源：arXiv AI · 关键词：#药物发现 #AIAgent #分子筛选 #MolClaw

### 可靠的 Agent 科学需要对抗实验（Sound Agentic Science Requires Adversarial Experiments）
LLM Agent 正在被快速用于科学数据分析，但这篇论文指出，这也在加速一种熟悉的失败模式：快速生产看似合理、可以无限修改的分析，本质上是把假设空间变成了为发表优化的候选声明。流畅的解释或单一数据集上的显著结果不等于验证。论文主张，可靠的 Agent 科学必须引入对抗实验——因为缺失的证据是一个负空间，只有刻意寻找反例才能真正检验假说。这是对 AI 加速科研热潮的一剂冷静提醒。

🔗 [原文链接](https://arxiv.org/abs/2604.22080) · 来源：arXiv AI · 关键词：#AI科学 #对抗实验 #LLMAgent #可靠性

### QuantClaw：为 OpenClaw 在关键处保持精度（QuantClaw: Precision Where It Matters for OpenClaw）
自主 Agent 系统因长上下文输入和多轮推理带来了显著的效率挑战，计算和资金成本在实际开发中可能高得令人却步。量化是降低成本和延迟的标准手段，但对 Agent 性能的影响在真实场景中尚不清楚。QuantClaw 分析了 OpenClaw 上多种复杂工作流的量化敏感度，发现精度需求高度依赖任务类型，据此提出了一个即插即用的精度路由插件，根据任务特征动态分配精度。轻量任务走低精度、关键任务保高精度，实用性很强。

🔗 [原文链接](https://arxiv.org/abs/2604.22577) · 来源：arXiv AI · 关键词：#量化 #Agent系统 #OpenClaw #效率优化

### 基于工件的 Agent 框架：面向自适应和可复现的医学影像处理（An Artifact-based Agent Framework for Adaptive and Reproducible Medical Image Processing）
医学影像研究正从受控基准评估转向真实临床部署，这对工作流的自适应性和可复现性提出了双重要求。这个框架引入语义层来增强医学影像处理，将中间和最终输出形式化为带有完整溯源记录的工件。Agent 可以根据数据集特定条件和不断变化的分析目标来配置工作流，同时保证所有转换和决策都被显式记录且可重新执行。对于医学 AI 从实验室走向临床的过程中面临的工程挑战，这是一个务实的解决方案。

🔗 [原文链接](https://arxiv.org/abs/2604.21936) · 来源：arXiv AI · 关键词：#医学影像 #AIAgent #工作流 #可复现性

## 🔧 开源项目

### 微软 VibeVoice：内置说话人分离的开源语音识别模型（microsoft/VibeVoice）
VibeVoice 是微软发布的 Whisper 风格语音转文字模型，MIT 许可证，亮点在于将说话人分离（speaker diarization）直接内置到模型中，而非作为后处理步骤。原始模型 VibeVoice-ASR 为 17.3GB，社区提供了 5.71GB 的 4-bit MLX 量化版本，可在 Mac 上通过 mlx-audio 一行命令运行。Simon Willison 实测处理一段播客音频耗时约 525 秒，提示处理速度约 50.7 tokens/sec。对于需要本地化、隐私友好的语音转写方案的开发者来说，这是一个值得尝试的选项。
