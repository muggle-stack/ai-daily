## 🚀 模型发布

### Meta 发布新模型 Muse Spark，meta.ai 聊天平台同步上线多项新工具（Meta's new model is Muse Spark, and meta.ai chat has some interesting tools）

Meta 今天发布了 Muse Spark，这是自去年 Llama 4 以来的首个新模型，但这次走的不是开源路线——模型仅以托管方式提供，API 目前处于面向少数用户的私有预览阶段。用户可以通过 meta.ai 体验该模型（需 Facebook 或 Instagram 账号登录），目前提供"Instant"和"Thinking"两种模式，未来还将推出支持更长推理时间的"Contemplating"模式。Meta 自报的基准测试显示，Muse Spark 在部分指标上与 Opus 4.6、Gemini 3.1 Pro 和 GPT 5.4 具有竞争力，但在 Terminal-Bench 2.0 上明显落后。Meta 也坦承在长程智能体系统和编程工作流方面仍有差距，正在持续投入。这是 Meta 首次以闭源托管形式发布主力模型，标志着其 AI 策略的一次重要转向。

## 📄 研究论文

### ClawsBench：在模拟工作空间中评估 LLM 生产力智能体的能力与安全性（ClawsBench: Evaluating Capability and Safety of LLM Productivity Agents in Simulated Workspaces）

LLM 智能体越来越多地被用于自动化邮件、日程、文档管理等生产力任务，但在真实服务上测试存在不可逆操作的风险。现有基准测试依赖简化环境，无法反映真实的、有状态的跨服务工作流。ClawsBench 提供了五个高保真模拟服务（Gmail、Slack、Google Calendar、Google Docs、Google Drive），支持完整的状态管理和确定性快照/恢复，包含 44 个结构化任务，覆盖单服务、跨服务和安全关键场景。这项工作填补了 LLM 智能体在真实办公场景中系统性评估的空白，对企业级 Agent 部署具有直接参考价值。

### ALTK-Evolve：AI 智能体的在岗学习方法（ALTK‑Evolve: On‑the‑Job Learning for AI Agents）

IBM Research 提出的 ALTK-Evolve 框架探索了 AI 智能体在实际部署过程中持续学习和自我进化的能力。不同于传统的离线训练范式，该方法让智能体在执行真实任务的同时进行"在岗学习"，逐步提升自身表现。这一思路对于需要长期运行、不断适应新场景的企业级 Agent 尤为关键。该研究发表在 HuggingFace 博客上，代表了 IBM 在智能体持续学习方向的最新探索。

### TRACE：面向能力缺陷的针对性智能体训练（TRACE: Capability-Targeted Agentic Training）

现有的智能体训练方法要么依赖与模型实际能力缺陷无关的合成数据，要么让模型在目标环境中隐式学习所有能力。TRACE 提出了一种端到端的环境特定智能体自我改进系统，通过对比成功与失败的执行轨迹，精准定位模型的能力短板，然后生成针对性的训练环境来补强。这种"哪里不会练哪里"的思路比大水漫灌式训练更高效，对提升 Agent 在特定场景下的可靠性很有启发。

🔗 [原文链接](https://arxiv.org/abs/2604.05336) · 来源：arXiv AI · 关键词：#智能体训练 #能力定向 #自我改进 #LLM

### PaperOrchestra：用于自动化 AI 论文写作的多智能体框架（PaperOrchestra: A Multi-Agent Framework for Automated AI Research Paper Writing）

PaperOrchestra 是一个多智能体框架，能将非结构化的研究素材转化为可提交的 LaTeX 论文，包括综合文献综述和自动生成的图表。同时推出了 PaperWritingBench，这是首个标准化的论文写作基准，基于 200 篇顶会论文逆向工程出原始素材。现有的自动写作系统通常与特定实验流程强耦合，且文献综述质量较浅，PaperOrchestra 试图在灵活性和质量上同时突破。对于 AI 辅助科研写作这个快速升温的方向，这项工作提供了一个系统性的评估框架。

### 不确定性引导的潜在诊断轨迹学习用于序贯临床诊断（Uncertainty-Guided Latent Diagnostic Trajectory Learning for Sequential Clinical Diagnosis）

临床诊断本质上是一个在不确定性下序贯获取证据的过程，但大多数基于 LLM 的诊断系统假设患者信息已完全可观测。该研究提出了潜在诊断轨迹学习（LDTL）框架，将序贯诊断建模为基于规划 LLM 智能体的决策过程，解决了临床数据集中缺乏显式诊断路径监督的难题。这项工作对于推动 AI 在真实临床场景中的落地具有重要意义，因为真实诊断从来不是一步到位的。

### IntentScore：面向计算机操作智能体的意图条件化动作评估（IntentScore: Intent-Conditioned Action Evaluation for Computer-Use Agents）

计算机操作智能体（CUA）在桌面环境中执行 GUI 操作时，缺乏对动作质量的评估机制，容易产生不可逆错误并级联传播。IntentScore 是一个计划感知的奖励模型，从跨三个操作系统的 39.8 万条离线 GUI 交互步骤中学习对候选动作进行评分，通过对比对齐和边际排序两个互补目标进行训练。该模型在架构上将每个候选动作的规划意图嵌入动作编码器，能区分动作相似但意图不同的情况，配对准确率达到 97.5%。这对于提升 GUI Agent 的可靠性和安全性是一个实用的技术方案。

🔗 [原文链接](https://arxiv.org/abs/2604.05157) · 来源：arXiv AI · 关键词：#计算机操作智能体 #GUI #奖励模型 #动作评估

### 从治理规范到可执行控制：智能体 AI 运行时护栏的分层转译方法（From Governance Norms to Enforceable Controls: A Layered Translation Method for Runtime Guardrails in Agentic AI）

智能体 AI 系统具备规划、工具使用、状态维护和多步执行等特性，其治理问题与单轮生成式 AI 有本质区别——风险在执行过程中涌现，而非仅在开发或部署阶段。ISO/IEC 42001、NIST AI RMF 等治理标准虽然高度相关，但本身不能直接产出可实施的运行时护栏。该论文提出了一种分层转译方法，将标准衍生的治理目标连接到四个控制层：治理目标、设计时约束、运行时中介和执行时护栏。随着 Agent 大规模部署临近，这类将合规要求落地为技术实现的工作变得越来越紧迫。

🔗 [原文链接](https://arxiv.org/abs/2604.05229) · 来源：arXiv AI · 关键词：#智能体AI #治理 #运行时护栏 #AI安全

### 突破值分解多智能体强化学习中的次优稳定点（Breakthrough the Suboptimal Stable Point in Value-Factorization-Based Multi-Agent Reinforcement Learning）

值分解是多智能体强化学习（MARL）中的主流范式，但其容易收敛到次优解的问题一直缺乏理论解释。该研究引入了"稳定点"这一新理论概念来刻画值分解在一般情况下的收敛行为，揭示了非最优稳定点是性能不佳的主要原因。论文进一步提出了通过迭代过滤非最优稳定点来逐步逼近最优解的算法方案。这项工作为 MARL 中一个长期存在的理论瓶颈提供了新的分析视角和解决思路。

🔗 [原文链接](https://arxiv.org/abs/2604.05297) · 来源：arXiv AI · 关键词：#MARL #值分解 #多智能体 #收敛性

### 技能图谱：面向大规模智能体技能的依赖感知结构化检索（Graph of Skills: Dependency-Aware Structural Retrieval for Massive Agent Skills）

当智能体的技能库扩展到数千个可复用技能时，将全部技能加载到上下文窗口会导致 token 成本飙升、幻觉增加和延迟上升。Graph of Skills（GoS）提出了一种推理时的结构化检索层，离线构建可执行的技能依赖图，推理时检索有界的相关技能子集。这种方法在保持任务完成率的同时大幅降低了上下文开销，对于构建实用的大规模 Agent 系统是一个很实际的工程优化方向。

🔗 [原文链接](https://arxiv.org/abs/2604.05333) · 来源：arXiv AI · 关键词：#智能体技能 #技能检索 #图结构 #LLM智能体

### TFRBench：评估预测系统推理能力的基准（TFRBench: A Reasoning Benchmark for Evaluating Forecasting Systems）

传统时间序列预测评估只关注数值精度，将基础模型当作黑箱。TFRBench 是首个专门评估预测系统推理能力的基准，考察模型对跨通道依赖、趋势和外部事件的分析能力。该基准横跨五个领域的十个数据集，并提出了一个多智能体框架通过迭代验证循环来合成数值锚定的推理轨迹。评估证实这些推理在因果层面是有效的，为预测系统从"能算对"走向"能说清为什么"提供了评估工具。

🔗 [原文链接](https://arxiv.org/abs/2604.05364) · 来源：arXiv AI · 关键词：#时间序列预测 #推理能力 #基准测试 #基础模型

### MMORF：用于设计多目标逆合成规划系统的多智能体框架（MMORF: A Multi-Agent Framework for Designing Multi-objective Retrosynthesis Planning Systems）

多目标逆合成规划需要在质量、安全性和成本之间动态平衡，MMORF 提供了一个模块化的多智能体框架来构建和评估不同的系统设计方案。框架包含可灵活组合的智能体组件，并在一个包含 218 个多目标逆合成任务的新基准上进行了验证。这项工作展示了 LLM 多智能体系统在化学领域的应用潜力，为药物发现和材料科学中的合成路线规划提供了新工具。

### MedGemma 1.5 技术报告（MedGemma 1.5 Technical Report）

Google 发布了 MedGemma 1.5 4B，在 MedGemma 1 基础上大幅扩展了医学影像能力：支持 CT/MRI 体积数据、病理全切片图像、边界框解剖定位、多时间点胸片分析，以及改进的医学文档理解（实验室报告、电子病历）。相比 MedGemma 1 4B，新版本在 3D MRI 疾病分类准确率上提升了 11%。所有这些模态被整合在单一架构中，通过长上下文 3D 体积切片和全切片病理采样等技术创新实现。作为一个 4B 参数的模型能覆盖如此广泛的医学影像模态，在效率和实用性上都值得关注。

### 更好的 Harness：用评估信号驱动 Agent Harness 爬坡优化的实践方法（Better Harness: A Recipe for Harness Hill-Climbing with Evals）

LangChain 团队分享了一个实用洞察：构建更好的 Agent 的关键不仅在于模型本身，更在于构建更好的 Harness（智能体的外部调度和编排层）。要自动化地改进 Harness，需要一个强学习信号来指导"爬坡"过程，他们选择用评估（evals）作为这个信号。文章详细介绍了具体的设计决策和实践经验，对于正在构建和迭代 Agent 系统的开发者来说，这是一份很接地气的工程指南。核心理念是：把 Agent 优化当作一个可度量、可迭代的工程问题来解决。
