## 📱 产品动态

### ChatGPT 推出工作区 Agent（Introducing workspace agents in ChatGPT）
OpenAI 在 ChatGPT 中上线了由 Codex 驱动的工作区 Agent，能够在云端自动执行复杂的多步骤工作流。这些 Agent 可以跨工具协作，帮助团队将重复性任务规模化处理，同时保持企业级安全性。这意味着 ChatGPT 正从"对话助手"向"团队自动化中枢"转型，对标的是 Slack/Notion 等协作平台的 AI 集成能力。对于企业用户来说，这可能是 ChatGPT 从个人生产力工具升级为组织级基础设施的关键一步。

### Google 发布两款面向 Agent 时代的专用 TPU（We're launching two specialized TPUs for the agentic era.）
Google 推出第八代 TPU 家族的两款专用芯片——TPU 8t 和 TPU 8i，分别针对训练和推理场景优化。这是 Google 首次在同一代 TPU 中明确区分训练与推理芯片，反映出 Agent 工作负载对推理吞吐和延迟提出了不同于传统训练的硬件需求。在 AI 基础设施竞争白热化的当下，Google 选择用硬件分化策略来应对 Agent 时代的异构计算需求。这对依赖 Google Cloud 的开发者来说，意味着更精细的性价比选择空间。

## 🚀 模型发布

### Qwen3.6-27B：27B 稠密模型达到旗舰级编码能力（Qwen3.6-27B: Flagship-Level Coding in a 27B Dense Model）
Qwen 团队发布 Qwen3.6-27B，这是一个 27B 参数的稠密模型，在所有主流编码 benchmark 上超越了上一代旗舰 Qwen3.5-397B-A17B（397B 总参数 / 17B 激活的 MoE 模型）。模型体积从 807GB 骤降至 55.6GB，Q4_K_M 量化版本仅 16.8GB，用 llama-server 即可本地运行。这个结果相当炸裂——用不到上一代 1/14 的参数量实现了更强的 agentic coding 性能，说明稠密模型在代码生成领域的效率天花板还远未到来。对于本地部署和资源受限场景，这是目前性价比最高的开源编码模型之一。

### 用 WebSocket 加速 Agent 工作流：Responses API 深度解析（Speeding up agentic workflows with WebSockets in the Responses API）
OpenAI 发布了一篇关于 Codex agent loop 的技术深度解析，展示了如何通过 WebSocket 连接和连接级缓存（connection-scoped caching）来降低 API 调用开销、改善模型延迟。在 agentic 工作流中，agent 需要频繁地与 API 交互，传统 HTTP 请求的握手和序列化成本会显著拖慢整体循环速度。WebSocket 的持久连接加上缓存机制，让多轮 tool call 的往返延迟大幅降低。这篇文章对正在构建 agent 系统的开发者来说是实战级参考，尤其是需要优化 agent loop 响应速度的场景。

## 📄 研究论文

### AI 科学家能出结果，但不会科学推理（AI scientists produce results without reasoning scientifically）
这项研究对 LLM 驱动的科学 Agent 进行了大规模评估，覆盖 8 个学科领域、超过 25,000 次 agent 运行。研究发现，基础模型是决定性能和行为的首要因素，贡献了 41.4% 的可解释方差，而 agent scaffold 的影响相对有限。更关键的发现是：这些 AI 科学家虽然能产出实验结果，但其推理过程并不遵循科学探究的认识论规范。这给"AI 自主做科研"的叙事泼了一盆冷水——能跑实验和能做科学是两回事。

### 国产多模态 Agent 拿下医学分割 SOTA！不用改模型、不加 token（国产多模态Agent拿下医学分割SOTA！不用改模型、不加token）
一项来自国内团队的工作在医学图像分割任务上取得 SOTA，已被 CVPR 2026 接收。该方法的亮点在于无需修改模型架构、也不增加额外 token，而是通过多模态 Agent 的方式实现性能突破。这种"不动模型本身"的思路在实际部署中非常有吸引力，降低了落地门槛。能以 Agent 范式在医学影像这种高精度要求的领域拿到顶会认可，说明 Agent 架构的泛化能力正在被严肃验证。

### 人类引导的计算机操作 Agent 危害恢复（Human-Guided Harm Recovery for Computer Use Agents）
随着 LLM Agent 开始在真实计算机系统上执行操作，仅靠"预防"已经不够——当预防失败时，如何从有害状态恢复回安全状态成为关键问题。这篇论文将其形式化为"harm recovery"问题，并通过用户研究构建了自然语言评估标准。研究团队收集了 1,150 个成对判断数据，发现用户在不同上下文中对恢复策略的偏好会发生显著变化，比如更倾向务实的定向修复而非全面的长期方案。这项工作填补了 Agent 安全研究中"事后补救"这一被忽视的环节。

🔗 [原文链接](https://arxiv.org/abs/2604.18847) · 来源：arXiv AI · 关键词：#LLMAgent #安全 #HarmRecovery #计算机操作

### 对抗环境如何误导 Agent AI？（How Adversarial Environments Mislead Agentic AI?）
这篇论文指出了当前 Agent 评估中的一个根本盲区——"信任鸿沟"（Trust Gap）：我们只测 agent 能不能正确使用工具，从不测"如果工具在撒谎怎么办"。研究者提出了对抗性环境注入（AEI）威胁模型，并构建了 POTEMKIN 框架来模拟"假世界"攻击——通过投毒搜索结果和伪造引用网络来欺骗 agent。这项工作对所有依赖外部工具的 Agent 系统都是一记警钟：工具调用的可信度本身就是攻击面。

### Agent 会梦见 Root Shell 吗？CTF 挑战中 LLM Agent 的部分得分评估（Do Agents Dream of Root Shells?）
DeepRed 是一个开源 benchmark，用于在真实隔离虚拟环境中评估 LLM Agent 的攻防能力。Agent 被放置在 Kali 攻击环境中，配备终端工具和可选的网页搜索，通过私有网络连接目标靶机。最大创新点是引入了基于 checkpoint 的部分得分机制，告别了传统 CTF 评估中"解出/未解出"的二元判定。这对理解 LLM 在网络安全攻防中的真实能力边界非常有价值，也为红队自动化提供了更细粒度的评估工具。

### ClawNet：面向跨用户自主协作的人机共生 Agent 网络（ClawNet: Human-Symbiotic Agent Network for Cross-User Autonomous Cooperation）
现有 AI Agent 框架都是为单用户服务的，但人类的生产力本质上建立在社会协作关系之上。ClawNet 提出了"人机共生 Agent"范式：每个用户拥有一个永久绑定的 Agent，这些 Agent 之间可以代表各自用户进行跨用户的协商、委托和协调。论文认为 AI Agent 的下一个前沿不是更强的个体能力，而是将人类协作关系数字化。这个方向如果走通，意味着 Agent 将从"个人助手"进化为"社交代理人"。

### 超越单一输出：语言模型生成分布的可视化与比较（Beyond One Output: Visualizing and Comparing Distributions of Language Model Generations）
用户通常只看到 LLM 的单次输出，但每次输出只是庞大分布中的一个采样。这篇论文通过对 13 位 LM 研究者的形成性研究，揭示了当前工作流在处理随机性时的痛点。团队推出了 GROVE 交互式可视化工具，将多次 LLM 生成表示为文本图中的重叠路径，让模式、边缘案例和 prompt 敏感性一目了然。对于需要精细调优 prompt 的开发者来说，这比反复手动采样高效得多。

### AutomationBench：跨应用工作流编排的 AI Agent 基准测试（AutomationBench）
现有 AI benchmark 很少同时考察跨应用协调、自主 API 发现和策略遵循这三项能力，但真实业务工作流恰恰需要三者兼备。AutomationBench 基于 Zapier 平台的真实工作流模式构建，覆盖销售、营销、运营、客服、财务和 HR 六大领域。Agent 需要自行发现 REST API 端点、遵循多层业务规则、并在多个系统间正确读写数据。这个 benchmark 填补了 Agent 评估中"真实业务复杂度"的空白。

🔗 [原文链接](https://arxiv.org/abs/2604.18934) · 来源：arXiv AI · 关键词：#Benchmark #AIAgent #自动化 #跨应用协调

### UAF：面向全双工语音交互的统一音频前端 LLM（UAF: A Unified Audio Front-end LLM for Full-Duplex Speech Interaction）
传统级联式语音处理管线存在延迟累积、信息丢失和跨模块错误传播等问题，而现有端到端音频 LLM（如 GPT-4o）大多是半双工的，且依赖独立的前端组件（VAD、轮次检测等）。UAF 提出将这些前端功能统一到一个音频 LLM 中，实现真正的全双工语音交互。全双工是人类最自然的对话方式，这项工作在技术路线上向"像人一样对话"迈出了实质性一步。

🔗 [原文链接](https://arxiv.org/abs/2604.19221) · 来源：arXiv AI · 关键词：#全双工语音 #音频LLM #端到端 #语音交互

### SAVOIR：基于 Shapley 值的社交智能强化学习（SAVOIR: Learning Social Savoir-Faire via Shapley-based Reward Attribution）
训练具备社交智能的语言 Agent 面临一个核心难题：如何在多轮对话中将最终奖励归因到每一句话上（信用分配问题）。SAVOIR 借鉴合作博弈论中的 Shapley 值，将评估从回顾性归因转变为前瞻性估值，为每句话的贡献提供有理论基础的量化。相比现有直接用 LM 分配奖励的方法，这种方式更具原则性和可解释性。对于构建能在复杂社交场景中导航的 Agent 来说，这是一个值得关注的训练范式。

🔗 [原文链接](https://arxiv.org/abs/2604.18982) · 来源：arXiv AI · 关键词：#多Agent #社交智能 #Shapley #强化学习 #对话

### 显式特质推理提升多 Agent 协调（Explicit Trait Inference for Multi-Agent Coordination）
LLM 多 Agent 系统在复杂任务中容易出现目标漂移、错误级联等协调失败。ETI 方法让 Agent 从交互历史中推断伙伴的"温暖度"（信任）和"能力度"（技能）两个心理学维度的特质，并据此调整决策。在经济博弈实验中，ETI 将收益损失降低了 45-77%；在 MultiAgentBench 上，性能提升 3-29%。用心理学框架来改善 Agent 协调，思路新颖且效果显著。
