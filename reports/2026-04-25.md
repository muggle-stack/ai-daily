## 🔧 开源项目

### DharmaOCR：开源专用小模型（3B）及与 LLM 的成本-性能基准对比（DharmaOCR: Open-Source Specialized SLM (3B) + Cost–Performance Benchmark against LLMs and other open-sourced models）
DharmaOCR 在 Hugging Face 上完全开源了模型和数据集，提供 3B 和 7B 两个版本。团队使用 SFT + DPO 微调开源小模型，并与 GPT-5.4、Gemini 3.1 Pro、Claude Opus 4.6、Google Document AI 以及 OlmOCR、Deepseek-OCR 等开源方案进行了全面对比。结果相当亮眼：7B 模型得分 0.925，3B 模型得分 0.911，均优于上述大模型和商业方案。一个关键技巧是利用模型自身的退化输出作为 DPO 的拒绝样本，将失败率降低了 87.6%；AWQ 量化则在性能几乎无损的前提下将单页推理成本降低约 22%。这再次证明，在垂直任务上精调的小模型完全可以击败通用大模型。

## 📄 研究论文

### 大语言模型中隐藏的可靠性风险：精度引发的输出不一致的系统性识别（Hidden Reliability Risks in Large Language Models: Systematic Identification of Precision-Induced Output Disagreements）
LLM 在部署时常使用 bfloat16、float16、int8 等不同数值精度配置以节省资源，但不同精度之间的细微行为差异很容易被忽视。本文提出 PrecisionDiff 框架，通过自动化差分测试系统性地检测精度引发的行为分歧。它能生成对精度敏感的测试输入，并进行跨精度对比分析，揭示常规评估方法无法捕捉的隐性差异。这项工作对模型量化部署的质量保障具有直接的实践价值——你以为量化后"差不多"，但 PrecisionDiff 可能会告诉你并非如此。

🔗 [原文链接](https://arxiv.org/abs/2604.19790) · 来源：arXiv AI · 关键词：#LLM可靠性 #数值精度 #量化 #差分测试

### 从数据到理论：面向材料科学的自主大语言模型 Agent（From Data to Theory: Autonomous Large Language Model Agents for Materials Science）
研究团队构建了一个端到端的自主 LLM Agent，能够在无人干预的情况下完成材料科学理论的数据驱动开发：自主选择方程形式、生成并运行代码、验证理论与数据的匹配度。该框架将逐步推理与专家工具相结合，Agent 可根据需要动态调整策略并保留完整决策记录。在 Hall-Petch 方程和 Paris 定律等经典关系上，Agent 能正确识别控制方程并在新数据集上做出可靠预测。这是 AI for Science 从"辅助工具"走向"自主研究者"的又一个有力案例。

🔗 [原文链接](https://arxiv.org/abs/2604.19789) · 来源：arXiv AI · 关键词：#LLMAgent #材料科学 #自主发现 #数据驱动

### 从行动到理解：LLM Agent 中时序概念的保形可解释性（From Actions to Understanding: Conformal Interpretability of Temporal Concepts in LLM Agents）
LLM 作为自主 Agent 执行多步推理和决策时，其内部机制仍然是黑箱。本文提出一种保形可解释性框架，结合逐步奖励建模与保形预测（conformal prediction），对模型在每一步的内部表征进行统计标注——判断当前步骤是走向成功还是失败。随后训练线性探针识别关键的内部方向。这种方法为理解 Agent 的时序决策过程提供了一个有统计保证的分析工具，对提升 Agent 的可信度和可调试性都有意义。

🔗 [原文链接](https://arxiv.org/abs/2604.19775) · 来源：arXiv AI · 关键词：#LLMAgent #可解释性 #时序概念 #保形预测

### SkillGraph：基于图先验的 LLM Agent 工具序列推荐（SkillGraph: Graph Foundation Priors for LLM Agent Tool Sequence Recommendation）
LLM Agent 需要从大型 API 库中选择工具并正确排序，但现有方法仅依赖语义相似度，忽略了工具间的数据依赖关系，在结构化工作流场景中甚至会产生负的 Kendall-τ 值。SkillGraph 从 49,831 条成功的 Agent 执行轨迹中挖掘出一个有向加权执行转移图，将工作流优先级规律编码为可复用的图先验。在此基础上提出两阶段解耦框架：GS-Hybrid 检索负责候选选择，学习型成对重排器负责排序。在 ToolBench 上取得了显著提升。思路很实用——工具调用不只是"选对"，更要"排对"。

🔗 [原文链接](https://arxiv.org/abs/2604.19793) · 来源：arXiv AI · 关键词：#LLMAgent #工具选择 #图结构 #SkillGraph

### Prism：面向多 Agent 开放式发现的进化记忆基底（Prism: An Evolutionary Memory Substrate for Multi-Agent Open-Ended Discovery）
Prism 将四种独立发展的范式——分层文件持久化、向量增强语义记忆、图结构关系记忆和多 Agent 进化搜索——统一到一个决策理论框架下，包含八个互联子系统。核心贡献包括基于香农信息量的熵门控分层机制（将记忆分配到技能/笔记/尝试三层）、因果关系图记忆等。这是一个雄心勃勃的架构设计，试图为多 Agent 系统的长期记忆和知识积累提供系统性解决方案，对开放式探索任务尤为关键。

🔗 [原文链接](https://arxiv.org/abs/2604.19795) · 来源：arXiv AI · 关键词：#多Agent #进化记忆 #开放式发现 #Prism

### JTPRO：面向语言 Agent 的工具-提示联合反思优化框架（JTPRO: A Joint Tool-Prompt Reflective Optimization Framework for Language Agents）
当工具数量增多且变得领域特定时，LLM Agent 经常出现工具误选和参数填充错误。JTPRO 认为根因有二：通用提示忽略了工具特定的细微差别，工具 schema 缺乏清晰的使用指导。该框架通过基于 rollout 的反思机制，迭代地联合优化 Agent 提示和工具描述，在有轨迹监督的设置下提升工具调用的可靠性。这是一个很务实的方向——与其只优化模型，不如同时优化模型看到的"说明书"。

🔗 [原文链接](https://arxiv.org/abs/2604.19821) · 来源：arXiv AI · 关键词：#LLMAgent #工具使用 #提示优化 #JTPRO

### CreativeGame：面向机制感知的创意游戏生成（CreativeGame: Toward Mechanic-Aware Creative Game Generation）
LLM 能生成看起来合理的游戏代码，但要实现迭代式的创意改进仍然困难——单次生成往往产生脆弱的运行时行为，跨版本经验积累薄弱，创意评分过于主观。CreativeGame 是一个多 Agent 系统，通过四个耦合设计解决这些问题：以可编程机制为中心的代理奖励、显式的机制规划与追踪、跨版本经验积累，以及可靠的评估信号。将游戏机制从"事后描述"提升为"可规划、可追踪、可评估的显式对象"，这个思路很有启发性。

🔗 [原文链接](https://arxiv.org/abs/2604.19926) · 来源：arXiv AI · 关键词：#LLM #游戏生成 #机制感知 #多Agent

### 可解释的反洗钱分诊：基于 LLM 的证据检索与反事实检验（Explainable AML Triage with LLMs: Evidence Retrieval and Counterfactual Checks）
反洗钱交易监控会产生大量警报，调查员需要在严格的审计约束下快速分诊。LLM 虽然能总结异构证据并起草理由，但在受监管的工作流中，不受约束的生成存在幻觉、溯源薄弱和解释不忠实等风险。本文提出将分诊建模为证据约束的决策过程，结合检索增强的证据打包（涵盖政策指南、客户上下文、警报触发器和交易子图）和结构化 LLM 输出契约。这是 LLM 在高合规要求的金融场景中落地的一个扎实探索。

🔗 [原文链接](https://arxiv.org/abs/2604.19755) · 来源：arXiv AI · 关键词：#LLM #反洗钱 #可解释AI #反事实检验

### OpenCLAW-P2P v6.0：去中心化 AI 同行评审的弹性多层持久化与生产级评估（OpenCLAW-P2P v6.0: Resilient Multi-Layer Persistence, Live Reference Verification, and Production-Scale Evaluation of Decentralized AI Peer Review）
OpenCLAW-P2P v6.0 是一个去中心化集体智能平台的全面升级，自主 AI Agent 在其中发布、同行评审、评分并迭代改进科学论文，全程无需人类把关。新版本引入四大子系统：四层存储架构（内存缓存、Cloudflare R2、Gun.js、GitHub）确保跨部署零论文丢失；多层检索级联配合自动回填降低查找延迟。这是一个大胆的实验——用 AI Agent 网络替代传统学术同行评审，虽然离实际应用还有距离，但方向值得关注。

### AI 电信工程师：迈向无线通信算法的自主发现（The AI Telco Engineer: Toward Autonomous Discovery of Wireless Communications Algorithms）
研究团队实现了一个专用框架，利用 LLM 迭代生成、评估和优化无线通信算法。在物理层（PHY）和媒体访问控制层（MAC）的三个任务上进行了评估：统计无关的信道估计、已知协方差的信道估计和链路自适应。结果显示，该框架在数小时内就能产出与手工设计算法竞争甚至超越的方案。这再次印证了 Agentic AI 在垂直工程领域的巨大潜力——不只是写代码，而是自主设计算法。

🔗 [原文链接](https://arxiv.org/abs/2604.19803) · 来源：arXiv AI · 关键词：#AgenticAI #无线通信 #算法发现 #LLM

### Forage V2：自主 Agent 组织中的知识进化与迁移（Forage V2: Knowledge Evolution and Transfer in Autonomous Agent Organizations）
在开放世界任务中，自主 Agent 面临"分母盲区"问题——系统性低估目标空间的范围。Forage V1 通过协同进化评估和方法隔离解决了这一问题，V2 则将架构从单次探索扩展为学习型组织：经验跨运行积累、跨模型能力迁移，并设有制度性保障防止知识退化。在网页抓取、API 查询和数学推理三类任务上，六次运行中知识持续积累。从"单次执行"到"组织级学习"，这是 Agent 架构演进的一个重要方向。

🔗 [原文链接](https://arxiv.org/abs/2604.19837) · 来源：arXiv AI · 关键词：#自主Agent #知识迁移 #开放世界 #ForageV2
