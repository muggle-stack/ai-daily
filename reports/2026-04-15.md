## 📄 研究论文

### LABBench2：面向生物学研究的 AI 系统改进基准（LABBench2: An Improved Benchmark for AI Systems Performing Biology Research）
AI 加速科学发现的热潮下，如何衡量 AI 在真实科研场景中的能力变得至关重要。LABBench2 是此前 LAB-Bench 的升级版，专注于评估 AI 系统在生物学研究中执行"有意义工作"的能力，而非仅停留在知识问答或推理层面。该基准将评估重心从"知道什么"转向"能做什么"，覆盖从专用基础模型到自主假设生成系统等多种 AI 应用形态。对于关注 AI for Science 赛道的研究者来说，这是一个值得跟踪的标尺。

🔗 [原文链接](https://arxiv.org/abs/2604.09554) · 来源：arXiv AI · 关键词：#LABBench2 #AI科学研究 #生物学 #Benchmark

### 屏幕上的图灵测试：移动端 GUI Agent 人类化基准（Turing Test on Screen: A Benchmark for Mobile GUI Agent Humanization）
自主 GUI Agent 越来越强，但数字平台的反检测机制也在升级——这篇论文首次将这场博弈形式化为一个 MinMax 优化问题。研究团队收集了高保真的移动端触控动力学数据集，发现基于大语言模型的 Agent 因运动学特征不自然而极易被识别。在此基础上，他们提出了 Agent Humanization Benchmark（AHB）和配套检测协议。这项工作揭示了一个被忽视的维度：Agent 要在真实人类生态中存活，"像人"可能比"能干"更关键。

🔗 [原文链接](https://arxiv.org/abs/2604.09574) · 来源：arXiv AI · 关键词：#GUIAgent #图灵测试 #人类化 #反检测

### 不等你开口就帮忙：已部署的主动式 Agent 工单支持系统（Help Without Being Asked: A Deployed Proactive Agent System for On-Call Support with Continuous Self-Improvement）
大规模云服务平台每天产生数千张工单，现有的 LLM Agent 通常只做"被动应答"——一旦问题升级给人工就退出了。Vigil 系统打破了这个模式，它是一个主动式 Agent，能在整个工单生命周期中持续参与：跟踪解决进度、协助后续问询，并从失败案例中自我学习。这种"不问就帮"的设计思路，对企业级 AI 客服的落地很有启发。

🔗 [原文链接](https://arxiv.org/abs/2604.09579) · 来源：arXiv AI · 关键词：#LLMAgent #主动式智能体 #云服务 #工单支持

### 用潜在基础模型进行 PDE 空间的 Agent 驱动探索（Agentic Exploration of PDE Spaces using Latent Foundation Models for Parameterized Simulations）
偏微分方程（PDE）描述的物理现象本质上是连续、高维且常常混沌的，传统探索依赖昂贵的实验或数值模拟。这篇论文的思路很巧妙：将多 Agent LLM 与潜在基础模型（LFM）耦合，后者在参数化模拟上学习紧凑、解耦的潜在表示。这样一来，LLM 就能像在药物发现领域操作离散 token 一样，对连续物理空间进行大规模自动化探索。对科学计算领域而言，这是一条将 LLM 能力延伸到连续域的可行路径。

🔗 [原文链接](https://arxiv.org/abs/2604.09584) · 来源：arXiv AI · 关键词：#PDE #基础模型 #科学计算 #Agent

### DeepReviewer 2.0：可追溯的 Agent 式科学同行评审系统（DeepReviewer 2.0: A Traceable Agentic System for Auditable Scientific Peer Review）
自动化同行评审不只是生成流畅的评语——审稿人和领域主席需要的是可审计的判断。DeepReviewer 2.0 围绕"输出契约"构建：生成带锚定标注、局部证据和可执行后续动作的可追溯评审包，只有满足最低可追溯性和覆盖预算后才允许导出。在 134 篇 ICLR 2025 投稿上的测试表明，这种过程控制的 Agent 架构显著提升了评审的透明度和可操作性。

🔗 [原文链接](https://arxiv.org/abs/2604.09590) · 来源：arXiv AI · 关键词：#自动同行评审 #Agent系统 #可追溯性

### Hubble：LLM 驱动的安全自动化 Alpha 因子发现框架（Hubble: An LLM-Driven Agentic Framework for Safe and Automated Alpha Factor Discovery）
量化金融中发现预测性 Alpha 因子一直是个难题——搜索空间巨大，信噪比极低。传统遗传编程方法容易产生复杂且不可解释的公式，还容易过拟合。Hubble 框架让 LLM 充当智能搜索启发式，受限于领域特定算子语言和基于 AST 的执行沙箱，通过 RankIC、年化信息比率等严格统计流水线评估候选因子。这是 LLM 在金融量化领域一个很实在的闭环应用。

🔗 [原文链接](https://arxiv.org/abs/2604.09601) · 来源：arXiv AI · 关键词：#LLM #量化金融 #Alpha因子 #Agent

### 与 AI 科学家竞赛：Agent 驱动的天体物理研究方法（Competing with AI Scientists: Agent-Driven Approach to Astrophysics Research）
研究团队将多 Agent 系统 Cmbagent（AI 科学家 Denario 的分析系统）应用于 FAIR Universe 弱引力透镜不确定性挑战赛，让专门化 Agent 协作完成研究构思、代码编写执行、结果评估和流水线迭代优化。纯自主探索初期未达到专家水平，但引入人机协作后性能显著提升。这个案例很好地展示了当前 AI 科学家的真实能力边界：自主性有了，但离完全替代人类专家还有距离。

🔗 [原文链接](https://arxiv.org/abs/2604.09621) · 来源：arXiv AI · 关键词：#AI科学家 #天体物理 #多Agent #参数推断

### OpenFlo：通过模拟人类 Web 交互实现自动化 UX 评估（OpeFlo: Automated UX Evaluation via Simulated Human Web Interaction with GUI Grounding）
传统 Web 可用性评估依赖耗时的用户研究和专家评审，严重拖慢产品迭代速度。OpenFlo 是一个 UX 评估 Agent，通过 GUI Grounding 直接与真实网页端到端交互，而非依赖 DOM 解析。系统结合模拟用户行为画像和结构化评估协议，集成了 SUS 量表和逐步单项易用性问卷。对小团队和敏捷工作流来说，这类工具的价值不言而喻。

🔗 [原文链接](https://arxiv.org/abs/2604.09581) · 来源：arXiv AI · 关键词：#UX评估 #GUIGrounding #WebAgent

### MobiFlow：基于轨迹融合的真实移动 Agent 基准测试（MobiFlow: Real-World Mobile Agent Benchmarking through Trajectory Fusion）
现有移动 Agent 评估基准（如 AndroidWorld）依赖系统级模拟器和 API 来判断任务是否成功，但真实场景中大量第三方应用并不暴露这些接口，导致评估与实际使用严重脱节。MobiFlow 提出了一种基于高效图构建算法的轨迹融合评估框架，支持从任意第三方应用中提取任务进行评测。这填补了移动 Agent 评估中一个很实际的空白。

🔗 [原文链接](https://arxiv.org/abs/2604.09587) · 来源：arXiv AI · 关键词：#移动Agent #Benchmark #GUI交互 #轨迹融合

### AI Agent 的持久身份：基于多锚点架构的弹性记忆与连续性（Persistent Identity in AI Agents: A Multi-Anchor Architecture for Resilient Memory and Continuity）
当前 AI Agent 有一个根本性问题：上下文窗口溢出、对话历史被压缩时，Agent 会经历灾难性遗忘——丢失的不仅是信息，还有自我的连续性。这篇论文借鉴人类记忆障碍的神经学案例，指出人类身份之所以能在损伤中存活，是因为它分布在情景记忆、程序记忆、情感连续性和具身知识等多个系统中。作者开源了 soul.py 架构，通过可分离组件实现持久身份。这个方向对长期运行的 Agent 系统设计很有参考价值。

🔗 [原文链接](https://arxiv.org/abs/2604.09588) · 来源：arXiv AI · 关键词：#AIAgent #持久身份 #记忆架构 #上下文窗口

### LLM 在部分可观测环境下的文本导航与探索（LLMs for Text-Based Exploration and Navigation Under Partial Observability）
这项研究测试了一个有趣的问题：LLM 能否在不执行代码、不使用工具的情况下，仅凭文本充当部分可观测环境中的控制器？实验在固定 ASCII 网格世界中进行，Agent 每步只能看到 5×5 的局部窗口，需要选择上下左右移动。9 个当代 LLM（涵盖开源/闭源、稠密/MoE、指令/推理调优）在探索和导航两个任务上接受了评测。结果揭示了 LLM 在空间推理方面的真实能力边界。

🔗 [原文链接](https://arxiv.org/abs/2604.09604) · 来源：arXiv AI · 关键词：#LLM #导航 #部分可观测 #空间推理

### DERM-3R：面向真实临床场景的资源高效多模态皮肤病诊疗 Agent 框架（DERM-3R: A Resource-Efficient Multimodal Agents Framework for Dermatologic Diagnosis and Treatment in Real-World Clinical Settings）
皮肤病影响全球数十亿人，而中医的辨证论治提供了互补的整体方法，但受限于知识非标准化、多模态记录不完整和专家推理难以规模化。DERM-3R 是一个资源高效的多模态 Agent 框架，在有限数据和算力条件下建模中医皮肤病诊疗流程。将中医智慧与现代 AI 架构结合，这个方向在国内临床场景中有独特的应用潜力。

🔗 [原文链接](https://arxiv.org/abs/2604.09596) · 来源：arXiv AI · 关键词：#多模态Agent #皮肤病诊断 #中医 #医疗AI

### 通用 LLM 作为人类驾驶行为模型：简化合流场景研究（General-purpose LLMs as Models of Human Driver Behavior: The Case of Simplified Merging）
人类行为模型对自动驾驶虚拟安全评估至关重要，但现有模型在可解释性和灵活性之间难以兼顾。这项研究将 OpenAI o3 和 Google Gemini 2.5 Pro 嵌入简化的一维合流场景作为闭环驾驶 Agent，与人类数据进行定量和定性对比。结果有助于理解通用 LLM 在驾驶行为建模上能捕捉什么、不能捕捉什么，为 LLM 在自动驾驶仿真中的应用划定了初步边界。

🔗 [原文链接](https://arxiv.org/abs/2604.09609) · 来源：arXiv AI · 关键词：#LLM #驾驶行为建模 #自动驾驶

### 超越机器人中的心智理论（Beyond Theory of Mind in Robotics）
心智理论（ToM）一直是机器人社会交互的主导范式，但这篇论文指出 ToM 的三个核心假设——意义从内部状态向外传递、理解需要旁观式推断、行为意义固定——都与真实社会交互的运作方式不符。作者借鉴民族方法学、会话分析和参与式意义建构理论，主张社会意义不是从行为中解码的，而是在 Agent 之间的逐时刻协调中产生的。这一视角对人机交互设计有深层启示。

🔗 [原文链接](https://arxiv.org/abs/2604.09612) · 来源：arXiv AI · 关键词：#心智理论 #机器人 #社会交互
