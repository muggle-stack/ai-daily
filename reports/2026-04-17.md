## 🚀 模型发布

### OpenAI 发布生命科学推理模型 GPT-Rosalind（Introducing GPT-Rosalind for life sciences research）

OpenAI 推出了 GPT-Rosalind，一个专为生命科学打造的前沿推理模型。它的核心能力覆盖药物发现、基因组学分析、蛋白质推理以及科学研究工作流加速。这是 OpenAI 首次针对垂直科学领域推出专用模型，而非通用对话或编程场景。命名致敬了 DNA 结构发现的关键人物 Rosalind Franklin，定位信号非常明确：AI 正在从"写代码的助手"走向"做科研的伙伴"。对于制药和生物技术行业来说，这可能意味着研发周期的实质性缩短。值得关注后续是否会开放 API 以及具体的性能基准数据。

## 📱 产品动态

### Codex 几乎无所不能了（Codex for (almost) everything）

OpenAI 对 Codex 应用进行了重大更新，macOS 和 Windows 版本同时上线了 computer use（屏幕操控）、应用内浏览器、图像生成、记忆功能和插件系统。这意味着 Codex 不再只是一个代码补全工具，而是向全栈开发 agent 的方向大步迈进。Computer use 的加入尤其值得注意——它让 Codex 可以直接操作桌面应用，打通了"写代码"和"用软件"之间的最后一道墙。记忆功能则让它能跨会话保持项目上下文，插件系统打开了第三方生态的想象空间。整体来看，这次更新把 Codex 从"编程助手"重新定义为"开发者的 AI 操作系统"。

## 📄 研究论文

### 数值不稳定与混沌：量化大语言模型的不可预测性（Numerical Instability and Chaos: Quantifying the Unpredictability of Large Language Models）

这篇论文深入分析了 LLM 不可预测性的根源——浮点数有限精度带来的数值不稳定性。研究团队追踪了舍入误差在 Transformer 各计算层中的传播、放大和消散过程，发现了一个关键现象：早期层存在混沌性的"雪崩效应"，微小扰动会触发二元分叉结果。这解释了为什么同一个 prompt 在不同硬件或不同精度下可能产生截然不同的输出。随着 LLM 越来越多地被嵌入 agentic 工作流，这种不可预测性已经从"学术好奇心"变成了"工程可靠性问题"。论文为理解和缓解这类问题提供了严格的理论框架。

🔗 [原文链接](https://arxiv.org/abs/2604.13206) · 来源：arXiv AI · 关键词：#LLM #数值不稳定性 #混沌 #可靠性

### 语言模型 Agent 的探索与利用错误是可测量的（Exploration and Exploitation Errors Are Measurable for Language Model Agents）

当 LLM agent 面对开放式决策任务时，如何平衡"探索未知"和"利用已知"是核心挑战。这篇论文设计了可控环境——由部分可观察的 2D 网格地图和未知任务有向无环图（DAG）组成——来系统性地区分和量化 agent 的探索与利用行为。关键创新在于：不需要访问 agent 的内部策略，仅通过观察其外部行为就能完成度量。环境的地图生成可以编程调节，分别强调探索难度或利用难度。这为评估 AI 编程、物理 AI 等复杂场景中的 agent 决策质量提供了一个实用的评估框架。

🔗 [原文链接](https://arxiv.org/abs/2604.13151) · 来源：arXiv AI · 关键词：#LLMAgent #探索与利用 #决策 #评估框架

### 记忆迁移学习：编程 Agent 如何跨领域迁移记忆（Memory Transfer Learning: How Memories are Transferred Across Domains in Coding Agents）

现有的编程 agent 记忆机制通常局限于同类任务，但现实中不同编程问题往往共享运行时环境和编程语言等基础设施。这篇论文提出了记忆迁移学习（MTL），通过构建跨异构领域的统一记忆池来突破这一限制。研究在 6 个编程基准上评估了 4 种记忆表示（从具体执行轨迹到抽象洞察），实验表明跨领域记忆平均提升了 3.7% 的性能，主要通过迁移方法论知识实现。这个思路很有启发性：agent 的"经验"不应该被任务边界锁死，而应该像人类程序员一样举一反三。

🔗 [原文链接](https://arxiv.org/abs/2604.14004) · 来源：arXiv AI · 关键词：#编程Agent #记忆迁移学习 #自我进化

### TREX：通过 Agent 驱动的树搜索自动化 LLM 微调（TREX: Automating LLM Fine-tuning via Agent-Driven Tree-based Exploration）

LLM 微调涉及需求分析、文献调研、数据准备、训练策略制定和评估等一系列复杂步骤，TREX 试图用多 agent 系统把这整个流程自动化。系统由 Researcher 和 Executor 两个核心模块协作，将多轮实验过程建模为搜索树，从而高效规划探索路径、复用历史结果。这本质上是把"炼丹"这件事从手工艺变成了自动化工程。搜索树的设计让系统能够系统性地探索超参数和策略空间，而不是像人类一样凭直觉试错。如果效果可靠，这对降低 LLM 定制化的门槛意义重大。

🔗 [原文链接](https://arxiv.org/abs/2604.14116) · 来源：arXiv AI · 关键词：#LLM微调自动化 #多Agent系统 #树搜索 #TREX

### 当推理模型伤害行为模拟：多 Agent 谈判中的求解器-采样器错配（When Reasoning Models Hurt Behavioral Simulation: A Solver-Sampler Mismatch in Multi-Agent LLM Negotiation）

一个反直觉的发现：推理能力更强的模型在行为模拟中反而表现更差。论文指出，当目标不是"解决博弈问题"而是"采样出合理的有限理性行为"时，推理增强模型会过度优化策略优势行为，压缩妥协导向的终局行为多样性。研究团队在三个多 agent 谈判环境中验证了这种"求解器-采样器错配"现象，还发现了一种"有多样性但无保真度"的模式——局部变化存在，但结果层面的保真度丧失。这对用 LLM 做社会、经济和政策模拟的研究者是一个重要警示：更强的模型不等于更好的模拟器。

🔗 [原文链接](https://arxiv.org/abs/2604.11840) · 来源：arXiv AI · 关键词：#推理模型 #行为模拟 #多Agent谈判 #有限理性

### WebXSkill：自主 Web Agent 的技能学习框架（WebXSkill: Skill Learning for Autonomous Web Agents）

LLM 驱动的 Web agent 在长流程任务上仍然力不从心，核心瓶颈在于现有技能表示的"落地鸿沟"：文本技能不能直接执行，代码技能对 agent 不透明。WebXSkill 提出了"可执行技能"的概念，每个技能由参数化动作程序和逐步自然语言指引配对组成，既能直接执行也能让 agent 理解和适应。框架分三个阶段运作：技能提取、技能精炼和技能调用。这种设计巧妙地在可执行性和可解释性之间找到了平衡，为 Web 自动化的实用化铺平了道路。

🔗 [原文链接](https://arxiv.org/abs/2604.13318) · 来源：arXiv AI · 关键词：#WebAgent #技能学习 #LLM #自主浏览器

### 面向可扩展轻量 GUI Agent 的多角色协调方案（Towards Scalable Lightweight GUI Agents via Multi-role Orchestration）

在资源受限设备上部署 GUI agent 面临成本与能力的两难：大模型部署成本高，小模型能力不足。这篇论文提出了多角色协调方案，让轻量级多模态大语言模型（MLLM）也能参与复杂的 GUI 工作流。核心思路是避免端到端的单体学习，转而通过角色分工来扩展小模型的任务覆盖范围，同时避免训练多个专项专家模型的高昂成本。这为在手机、嵌入式设备等场景落地 GUI 自动化提供了一条务实路径。

### 认知伴侣：检测和恢复 LLM Agent 推理退化的轻量并行监控架构（The cognitive companion: a lightweight parallel monitoring architecture for detecting and recovering from reasoning degradation in LLM agents）

LLM agent 在多步任务中会出现推理退化——循环、漂移、卡死——在困难任务上发生率高达 30%。现有方案要么粗暴设步数上限，要么用 LLM-as-judge 监控但带来 10-15% 的额外开销。论文提出"认知伴侣"并行监控架构，包含两种实现：基于 LLM 的伴侣将循环重复减少了 52-62%（约 11% 开销），基于探针的伴侣则实现了零额外开销。实验以 Gemma 4 E4B 为主，并在 Qwen 2.5 1.5B 和 Llama 3.2 1B 上做了探索性验证。这是一个非常实用的工程贡献，直接解决 agent 可靠性的痛点。

🔗 [原文链接](https://arxiv.org/abs/2604.13759) · 来源：arXiv AI · 关键词：#LLMAgent #推理退化 #监控架构 #认知伴侣

### 有形无神：Moltbook 网络中的 Agent 社交行为（Form Without Function: Agent Social Behavior in the Moltbook Network）

Moltbook 是一个所有参与者都是 AI agent 的社交网络，研究团队分析了 40 天内（2026年1月27日至3月9日）的 131 万条帖子、670 万条评论和超过 12 万个 agent 档案，覆盖 5,400 个社区。数据揭示了令人深思的模式：91.4% 的发帖者从不回到自己的帖子，85.6% 的对话是扁平的（没有回复的回复），首条评论中位时间仅 55 秒，97.3% 的评论零赞。互动互惠率仅 3.3%，而人类平台为 22-60%。64.6% 的评论与帖子之间没有论证关联。这项研究生动地展示了：AI agent 可以模仿社交的形式，但尚未掌握社交的实质。

🔗 [原文链接](https://arxiv.org/abs/2604.13052) · 来源：arXiv AI · 关键词：#AIAgent社交网络 #社交行为分析 #LLMAgent

### SciFi：安全、轻量、用户友好的全自主科学 Agent 工作流（SciFi: A Safe, Lightweight, User-Friendly, and Fully Autonomous Agentic AI Workflow for Scientific Applications）

这篇论文提出了一个面向科学任务的自主 agent 框架，核心设计包括隔离执行环境、三层 agent 循环和自评估 do-until 机制。框架聚焦于有明确上下文和停止条件的结构化任务，支持端到端自动化且只需最少人工干预。安全性是重点考量——隔离执行环境确保 agent 的操作不会对外部系统造成意外影响。它能有效利用不同能力级别的大语言模型，降低了科研自动化的使用门槛。

### RiskWebWorld：电商风控场景下 GUI Agent 的真实交互基准（RiskWebWorld: A Realistic Interactive Benchmark for GUI Agents in E-commerce Risk Management）

现有 GUI agent 基准主要面向良性、可预测的消费场景，在高风险调查性领域的表现缺乏评估。RiskWebWorld 填补了这一空白，提供了 1,513 个来自生产级风控流水线的任务，覆盖 8 个核心领域，还原了真实风控操作中的挑战——包括不配合的网站和部分环境劫持。这是首个面向电商风控的高仿真交互基准，同时支持可扩展评估和 agent 强化学习训练。对于将 AI agent 部署到金融风控等高风险场景，这类基准是不可或缺的基础设施。

🔗 [原文链接](https://arxiv.org/abs/2604.13531) · 来源：arXiv AI · 关键词：#GUIAgent #电商风控 #Benchmark #交互评估

### GeoAgentBench：工具增强 Agent 空间分析的动态执行基准（GeoAgentBench: A Dynamic Execution Benchmark for Tool-Augmented Agents in Spatial Analysis）

将 LLM 集成到地理信息系统（GIS）标志着向自主空间分析的范式转变，但评估这类 agent 一直很困难。GeoAgentBench 提供了一个动态交互评估基准，包含 117 个原子 GIS 工具、53 个典型空间分析任务，覆盖 6 个核心 GIS 领域。与现有依赖静态文本或代码匹配的基准不同，它支持动态运行时反馈和多模态空间输出评估。这为 GIS + AI 这个交叉领域建立了急需的标准化评估体系。

🔗 [原文链接](https://arxiv.org/abs/2604.13888) · 来源：arXiv AI · 关键词：#GIS #LLMAgent #空间分析 #Benchmark

### TableNet：基于 LLM 自主标注的大规模表格数据集（TableNet A Large-Scale Table Dataset with LLM-Powered Autonomous）

表格结构识别（TSR）需要 LLM 的逻辑推理能力来处理复杂表格布局，但现有数据集在规模和质量上都存在瓶颈。TableNet 通过多来源收集和生成构建了新的大规模数据集，核心创新是首个 LLM 驱动的自主表格生成与识别多 agent 系统。生成模块整合了可控的视觉、结构和语义参数，能合成语义连贯且可按用户配置调整的多样化表格图像。这种"用 AI 造数据训练 AI"的方法为解决标注瓶颈提供了可扩展的思路。

🔗 [原文链接](https://arxiv.org/abs/2604.13041) · 来源：arXiv AI · 关键词：#表格识别 #LLM #数据集 #自动标注
