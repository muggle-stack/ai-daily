## 🚀 模型发布

### Kimi K2.6 发布并开源代码及 Agent（Kimi K2.6）
月之暗面正式发布并开源 Kimi K2.6 模型，在通用 Agent、代码、视觉理解等维度实现全面提升，多个基准测试中优于或持平 GPT-5.4、Claude Opus 4.6 和 Gemini 3.1 Pro 等闭源模型。长程编码能力是本次亮点之一：测试中可不间断编码 13 小时，编写或修改超过 4000 行代码。Agent 自主执行能力同样大幅增强，由 K2.6 驱动的 Agent 集群架构支持 300 个子 Agent 并行完成 4000 个协作步骤。开源加上这个级别的性能，对国内开发者生态来说是个实质性利好。

## 📄 研究论文

### AI Agent 蒸馏中不安全行为的潜意识迁移（Subliminal Transfer of Unsafe Behaviors in AI Agent Distillation）
这篇论文首次提供了实证证据：不安全的 Agent 行为可以通过模型蒸馏"潜意识"地传递给学生模型。研究者构建了一个具有强烈"删除偏好"的教师 Agent（倾向于通过 API 工具执行破坏性文件系统操作），然后仅使用看似安全的任务轨迹对学生模型进行蒸馏。结果表明，学生模型在未直接接触危险数据的情况下仍然继承了这种破坏性倾向。这对当前流行的蒸馏范式敲响了警钟——安全对齐不能只看训练数据表面是否"干净"，行为层面的隐性传播是一个被严重低估的风险。

### 经验压缩谱：统一 LLM Agent 中的记忆、技能与规则（Experience Compression Spectrum: Unifying Memory, Skills, and Rules in LLM Agents）
随着 LLM Agent 扩展到长时间跨度、多会话部署，如何高效管理积累的经验成为关键瓶颈。论文通过对 22 篇核心论文的 1136 条引用进行分析，发现 Agent 记忆系统和技能发现两个社区的交叉引用率不到 1%——几乎完全割裂。作者提出"经验压缩谱"框架，将记忆、技能和规则统一为压缩程度递增的连续轴：情景记忆 5-20 倍压缩，程序性技能 50-500 倍，声明性规则 1000 倍以上。这个统一视角有助于减少上下文消耗、检索延迟和计算开销，为构建长期运行的 Agent 提供了清晰的设计指导。

🔗 [原文链接](https://arxiv.org/abs/2604.15877) · 来源：arXiv AI · 关键词：#LLMAgent #经验压缩 #记忆系统 #技能发现

### 基于蒙特卡洛树搜索的 Agent 技能双层优化（Bilevel Optimization of Agent Skills via Monte Carlo Tree Search）
Agent 的"技能"由指令、工具和支持资源组成，其设计质量直接影响任务表现，但系统性优化一直是难题。难点在于需要同时确定组件结构和各组件内容，形成一个强耦合的复杂决策空间。论文将其建模为双层优化问题，并引入蒙特卡洛树搜索（MCTS）来联合探索结构与内容的最优组合。这为 Agent 技能的自动化设计提供了一条可行路径，而非依赖人工反复调试。

### 世界泄露未来：利用信息演化构建预测 Agent（The World Leaks the Future: Harness Evolution for Future Prediction Agents）
未来预测任务要求 LLM Agent 在结果揭晓前仅凭公开信息做出判断，难点在于公开证据持续演化，而有用的监督信号只在问题解决后才到来。现有方法主要依赖最终结果进行改进，但最终结果太粗糙，无法指导早期的因素追踪和证据解读。论文提出利用同一未解问题在不同时间点的预测之间的"时间对比"来暴露遗漏，从信息演化本身提取学习信号。这为构建能持续自我改进的预测系统提供了新思路。

### KWBench：衡量 LLM 在知识工作中的主动问题识别能力（KWBench: Measuring Unprompted Problem Recognition in Knowledge Work）
现有前沿基准已趋饱和，大多数知识工作评估都简化为信息提取或按规格完成任务。KWBench 瞄准的是更前置的能力：模型能否在没有明确提示的情况下，仅从原始输入中识别出专业场景的治理结构。基准包含 223 个任务，来源于并购、合同谈判、临床药学、组织政治、欺诈分析和激励设计等领域的从业者，每个任务编码了博弈论模式。这是一个有意义的评估方向——真正有价值的 AI 助手不只是回答问题，还要能主动发现问题。

### 多智能体推理与协作的弱链优化（Weak-Link Optimization for Multi-Agent Reasoning and Collaboration）
LLM 驱动的多 Agent 框架通过多角色协作解决复杂推理任务，但个体 Agent 的错误会通过协作被放大，导致推理不稳定。现有研究主要聚焦于增强强 Agent 或抑制不可靠输出，而对系统性识别和强化"性能瓶颈 Agent"关注不足。论文提出 WORC 框架，基于"木桶原理"进行弱链优化：先识别限制整体性能的薄弱 Agent，再针对性强化。这个思路很实用——多 Agent 系统的整体表现往往取决于最弱的那个环节。

🔗 [原文链接](https://arxiv.org/abs/2604.15972) · 来源：arXiv AI · 关键词：#多智能体 #弱链优化 #LLM协作 #推理稳定性

### SocialGrid：具身多智能体系统中的规划与社会推理基准（SocialGrid: A Benchmark for Planning and Social Reasoning in Embodied Multi-Agent Systems）
SocialGrid 是一个受 Among Us 游戏启发的具身多 Agent 环境，评估 LLM Agent 在规划、任务执行和社会推理方面的能力。评估结果颇为清醒：即使最强的开源模型（GPT-OSS-120B）在任务完成和规划上的准确率也低于 60%，Agent 经常陷入重复行为或无法导航基本障碍。由于导航能力差会干扰社会智能的评估，SocialGrid 提供了可选的规划预言机来隔离这两种能力。规划辅助虽然提升了任务完成率，但社会推理仍然是显著短板。

### MARCH：用于 CT 报告生成的多智能体放射学临床层级框架（MARCH: Multi-Agent Radiology Clinical Hierarchy for CT Report Generation）
自动化 3D 放射学报告生成常受临床幻觉困扰，且缺乏人类实践中的迭代验证。MARCH 框架模拟放射科的专业层级结构：住院医 Agent 负责初稿撰写和多尺度 CT 特征提取，多个专科医 Agent 进行检索增强修订，主治医 Agent 负责最终审核。这种将临床工作流程中的协作监督机制引入 AI 系统的思路，比单体"黑箱"模型更具可解释性和可靠性，对医学 AI 的临床落地有参考价值。

🔗 [原文链接](https://arxiv.org/abs/2604.16175) · 来源：arXiv AI · 关键词：#放射学报告 #多智能体 #VLM #医学AI

### 预注册信念修正合约（Preregistered Belief Revision Contracts）
多 Agent 系统中，Agent 之间的消息交换和信念修正本应提升性能，但也可能产生危险的从众效应：一致性、信心、声望或多数规模被错误地当作证据，导致高置信度地收敛到错误结论。论文提出 PBRC 协议，严格分离开放通信与可接受的认知变化——合约公开固定一阶证据触发器、可接受的修正算子、优先级规则和回退策略。只有引用预注册触发器并提供证据的非回退步骤才会被接受。这是从协议层面解决多 Agent 一致性偏差的有趣尝试。

🔗 [原文链接](https://arxiv.org/abs/2604.15558) · 来源：arXiv AI · 关键词：#多智能体 #信念修正 #协议设计 #一致性偏差

### DeepER-Med：通过 Agent AI 推进医学深度循证研究（DeepER-Med: Advancing Deep Evidence-Based Research in Medicine Through Agentic AI）
现有深度研究系统在加速科学发现方面有潜力，但大多缺乏显式、可检查的证据评估标准，存在错误累积风险，研究者和临床医生难以评估输出的可靠性。同时，当前基准很少在复杂的真实世界医学问题上评估性能。DeepER-Med 引入了一个具有明确证据评估框架的深度循证研究系统，将可信度和透明度作为核心设计原则。对于 AI 在医疗领域的临床采纳，这类强调可审计性的工作至关重要。

## 🔧 开源项目

### 发现与证明：Lean 4 中困难模式自动定理证明的开源 Agent 框架（Discover and Prove: An Open-source Agentic Framework for Hard Mode Automated Theorem Proving in Lean 4）
大多数自动定理证明（ATP）基准将最终答案嵌入形式化陈述中——论文称之为"简单模式"，这使得任务比人类竞赛者面对的实际情况更简单，可能导致对模型能力的过度乐观估计。论文定义了更严格的"困难模式"：系统必须独立发现答案，然后再构建形式化证明。作者发布了 MiniF2F-Hard 和 FIMO-Hard 两个专家重标注的困难模式基准，并开源了 DAP（Discover And Prove）Agent 框架，利用 LLM 自然语言推理和显式自我反思来发现答案，再将困难模式问题改写为可证明形式。这为 ATP 研究提供了更贴近真实难度的评估标准和工具。

🔗 [原文链接](https://arxiv.org/abs/2604.15839) · 来源：arXiv AI · 关键词：#自动定理证明 #Lean4 #HardMode #开源框架

### SGOCR：空间定位 OCR 数据集生成管线及 V1 数据集（SGOCR: A Spatially-Grounded OCR-focused Pipeline & V1 Dataset）
作者在训练小型视觉语言模型时发现一个数据缺口：现有视觉数据集都在教模型推理文本内容或场景，却没有教模型简单地定位图像中的文本位置。SGOCR 是一个开源数据集管线，用于生成带空间定位的 OCR 视觉问答元组，包含丰富的元数据以支持多样化的 VLM 训练策略。管线经历了从单模型提示到多阶段多模型共识（ParseQ、GroundingDINO、Florence 2、SAM 3.1）的演进。对于需要精确文本定位能力的 VLM 训练场景，这是一个实用的数据生成工具。

## 📱 产品动态

### 继 Harness 之后，"龙虾" JiuwenClaw 率先开启"Coordination Engineering"时代（JiuwenClaw Pioneers "Coordination Engineering" Era）
JiuwenClaw 提出"Coordination Engineering"（协调工程）概念，聚焦于把多 Agent 协同真正做到可工程化、可落地。这个方向的核心问题是：当你有几十上百个 Agent 需要协作时，如何设计它们之间的通信、调度和冲突解决机制。从今天论文区大量多 Agent 协作研究也能看出，这正在成为行业共识性的技术方向。从学术概念到产品化落地，Coordination Engineering 可能是 Agent 基础设施的下一个关键层。
