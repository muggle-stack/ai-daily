## 📄 研究论文

### Signals：无需 LLM 评判，找到最有信息量的 Agent 轨迹（Signals: finding the most informative agent traces without LLM judges）

Katanemo Labs（DigitalOcean 旗下）发布了针对 Agentic 系统的新研究 Signals。核心问题是：构建 Agent 时产生的轨迹（traces）数量庞大，逐条人工或 LLM 审查成本极高。该方法提出一种轻量级结构化"信号"计算方式，能从实时 Agent 交互中自动筛选出最值得关注的轨迹，且不改变 Agent 的在线行为，也不需要 GPU。信号按交互、执行、环境三类模式分组，涵盖对齐偏差、停滞、脱离、失败等维度。对于正在大规模部署 Agent 的团队来说，这是一个实用的可观测性工具方向。

### 浙大校友用 AI 突破 32 年拉姆齐数下界（浙大校友用AI突破32年拉姆齐数下界）

浙大校友利用 AI 方法将拉姆齐数 R(3,17) 的下界从 92 提升至 93，打破了该问题 32 年未被改进的纪录。拉姆齐数是组合数学中的经典难题，其计算复杂度极高，传统方法长期停滞。这一突破展示了 AI 在纯数学领域辅助发现新结果的潜力，也是"AI for Math"方向的又一个标志性案例。

### 澄清行为选择模型的角色（Clarifying the role of the behavioral selection model）

这篇来自 Alignment Forum 的文章是对此前"行为选择模型预测 AI 动机"一文的补充说明。核心观点是：训练中非常相似甚至相同的行为，可能对应部署时截然不同的结果，关键在于驱动行为的"动机"不同。作者认为行为选择模型在中短期内具有较强预测力，但遗漏了一些预测 AI 动机的重要动态因素。对于关注 AI 安全和对齐的研究者来说，这篇文章有助于厘清"行为相同≠动机相同"这一容易被忽视的区分。

### MachinaCheck：基于 AMD MI300X 的多智能体 CNC 可制造性分析系统（MachinaCheck: Building a Multi-Agent CNC Manufacturability System on AMD MI300X）

MachinaCheck 是一个在 AMD MI300X GPU 上构建的多智能体系统，用于分析 CNC 加工零件的可制造性。该项目来自 lablab.ai 与 AMD 联合举办的开发者黑客松。系统通过多个 Agent 协作，对零件设计进行制造约束检查，帮助工程师在设计阶段就发现加工难点。这是多智能体架构在工业制造领域的一个具体落地案例，也展示了 MI300X 在推理密集型工作负载上的应用潜力。

### Parax v0.7：JAX 中的参数化建模库（Parax v0.7: Parametric Modeling in JAX）

Parax 是一个基于 JAX 的参数化建模库，试图在纯 JAX PyTrees 和面向对象建模方式（如 Equinox）之间架起桥梁。v0.7 版本带来了更精炼的 API，新增了有界优化（JAXopt）和贝叶斯采样（BlackJAX）的示例。支持派生/约束参数、可计算 PyTrees、可调用参数化，以及固定、有界和概率参数的抽象接口。适合需要在 JAX 生态中做结构化参数管理的研究者。
