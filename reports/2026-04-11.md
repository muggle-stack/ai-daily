### Riemann-Bench：面向"登月级"数学的基准测试（Riemann-Bench: A Benchmark for Moonshot Mathematics）

当前 AI 在国际数学奥林匹克上已经能拿金牌，但竞赛数学只是数学推理的一小片天地——题目领域有限、对高等数学工具要求不高，有时靠巧妙技巧就能得分。Riemann-Bench 由常春藤盟校数学教授、博士生和 IMO 奖牌得主联合出题，包含 25 道专家精选的研究级数学问题，每道题的作者本人都花了数周才解出。这个基准把评估标杆从"竞赛解题"推向了"真正的数学研究能力"，对当前所有前沿模型都是严峻挑战。如果你关心 LLM 的推理天花板在哪里，这篇论文给出了一个更诚实的标尺。

### 链式推理，树式学习：多轮 Agent 策略优化的自我纠正与嫁接（Reason in Chains, Learn in Trees: Self-Rectification and Grafting for Multi-turn Agent Policy Optimization）

LLM Agent 做多步推理时，强化学习的奖励信号往往非常稀疏，现有方法（如 GRPO）把采样轨迹当作独立链条，对每一步给予均匀的信用分配，忽略了某些关键步骤对最终结果的决定性影响。T-STAR 框架提出将多条轨迹合并为一棵"认知树"，通过识别功能相似的步骤节点来还原轨迹之间隐含的关联奖励结构。在此基础上引入"内省估值"机制，让 Agent 能更精准地定位和纠正关键决策点。这种从链到树的视角转换，为多轮推理任务中的信用分配问题提供了一个结构化的新思路。

🔗 [原文链接](https://arxiv.org/abs/2604.07165) · 来源：arXiv AI · 关键词：#T-STAR #强化学习 #多步推理 #Agent优化

### 自我修正的 Agent 到底需要多少 LLM？（How Much LLM Does a Self-Revising Agent Actually Need?）

现在的 LLM Agent 把世界建模、规划和反思全塞进一个语言模型循环里，效果不错但带来一个根本性的科学问题：Agent 的能力到底有多少来自 LLM 本身，又有多少来自外部的显式结构？这篇论文没有试图给出通用答案，而是设计了一套"声明式反思运行时协议"，把 Agent 的状态、置信信号、受保护动作和假设性转换全部外化为可检查的运行时结构。他们在带噪声的协作战舰游戏上用四种递进结构进行了实验，结果表明相当一部分"智能行为"其实可以归因于结构设计而非 LLM 能力。对于 Agent 架构设计者来说，这是一个很有价值的解耦视角。

🔗 [原文链接](https://arxiv.org/abs/2604.07236) · 来源：arXiv AI · 关键词：#LLM_Agent #能力归因 #结构化推理 #自我修正

### Qualixar OS：面向 AI Agent 编排的通用操作系统（Qualixar OS: A Universal Operating System for AI Agent Orchestration）

Qualixar OS 自称是首个应用层级别的通用 AI Agent 编排操作系统，与内核级方案（AIOS）或单框架工具（AutoGen、CrewAI）不同，它为异构多智能体系统提供完整运行时，支持 10 个 LLM 提供商、8+ 个 Agent 框架和 7 种传输协议。核心贡献包括：12 种多智能体拓扑的执行语义（网格、森林、网状等）、基于 LLM 的团队设计引擎 Forge、结合 Q-learning 与贝叶斯 POMDP 的三层模型路由，以及带 Goodhart 检测和 JSD 漂移监控的共识评判管线。如果多智能体系统要走向工程化落地，这类"操作系统级"的基础设施探索是绕不开的一步。

🔗 [原文链接](https://arxiv.org/abs/2604.06392) · 来源：arXiv AI · 关键词：#多智能体编排 #AgentOS #多智能体系统

### 小语言模型 Agent 的情感敏感决策研究（On Emotion-Sensitive Decision Making of Small Language Model Agents）

小语言模型（SLM）越来越多地被用作交互式决策 Agent，但现有评估几乎都忽略了情感作为因果因素对行为的影响。这项研究用 activation steering 技术在表征层面诱导情感状态，而非简单的 prompt 注入，实现了可控且可迁移的情感干预。评估基准围绕博弈论中的经典决策模板构建，涵盖合作与竞争激励、完全与不完全信息等场景。研究发现情感状态确实会系统性地影响 SLM 的决策行为，这对安全关键场景下的模型部署有直接的警示意义。

🔗 [原文链接](https://arxiv.org/abs/2604.06562) · 来源：arXiv AI · 关键词：#小语言模型 #情感决策 #ActivationSteering #博弈论

### KD-MARL：多智能体强化学习中的资源感知知识蒸馏（KD-MARL: Resource-Aware Knowledge Distillation in Multi-Agent Reinforcement Learning）

多智能体强化学习（MARL）在真实部署时受限于计算、内存和推理时间，专家策略虽然性能好但对边缘设备来说太重了。现有的知识蒸馏方法在 MARL 场景中只关注动作模仿，忽略了协调结构，还假设所有 Agent 能力一致。KD-MARL 提出一个两阶段框架，将集中式专家的协调行为迁移到轻量级去中心化 Agent 上，同时感知各 Agent 的资源约束。这项工作为 MARL 系统在嵌入式平台和边缘设备上的实际部署提供了一条可行路径。

🔗 [原文链接](https://arxiv.org/abs/2604.06691) · 来源：arXiv AI · 关键词：#知识蒸馏 #MARL #边缘部署 #多智能体

### AgentGate：面向"Agent 互联网"的轻量级结构化路由引擎（AgentGate: A Lightweight Structured Routing Engine for the Internet of Agents）

随着 AI Agent 系统快速发展，一个"Agent 互联网"正在浮现——专业化 Agent 分布在本地设备、边缘节点、私有服务和云平台上。虽然 Agent 的命名、发现和交互已有不少进展，但在延迟、隐私和成本约束下的高效请求分发仍是开放问题。AgentGate 将路由建模为受约束的决策问题而非自由文本生成，分解为动作决策和结构化落地两个阶段。这种设计在保持轻量的同时，能感知候选 Agent 的能力进行精准分发，是多智能体基础设施中一块重要的拼图。

### EmoMAS：面向高风险边缘部署谈判的情感感知多智能体系统（EmoMAS: Emotion-Aware Multi-Agent System for High-Stakes Edge-Deployable Negotiation with Bayesian Orchestration）

LLM 在自动化谈判中已被广泛使用，但高计算成本和隐私风险限制了在移动助手、救援机器人等隐私敏感的端侧场景中的部署。EmoMAS 是一个贝叶斯多智能体框架，将情感决策从被动反应转变为主动策略，由贝叶斯编排器协调博弈论、强化学习和心理一致性三个专业化 Agent。系统实时融合三方洞察来优化情感状态转换，同时持续更新对手模型。这项工作把小语言模型、情感建模和多智能体架构结合在一起，瞄准的是端侧高风险谈判这个很有实际价值的场景。

### EVGeoQA：动态多目标地理空间探索的 LLM 基准测试（EVGeoQA: Benchmarking LLMs on Dynamic, Multi-Objective Geo-Spatial Exploration）

现有的地理空间问答（GSQA）基准大多聚焦于静态检索，无法反映真实世界中涉及动态用户位置和复合约束的规划复杂性。EVGeoQA 以电动汽车充电场景为载体，每个查询都绑定用户的实时坐标，并整合充电需求和协同定位两个目标，形成了一种"位置锚定+双目标"的独特设计。这个基准测试能有效检验 LLM 在动态地理环境中的推理和规划能力，对自动驾驶、物流等领域的 AI 应用有参考价值。

### 重新审视推理 SFT 的泛化能力：优化、数据与模型能力的条件分析（Rethinking Generalization in Reasoning SFT: A Conditional Analysis on Optimization, Data, and Model Capability）

"SFT 只会记忆，RL 才能泛化"——这个说法在 LLM 后训练领域几乎成了共识。这篇论文用长链式思维（CoT）监督的推理 SFT 重新检验了这一论断，发现跨域泛化并非不存在，而是有条件的，由优化动态、训练数据和基座模型能力共同决定。一个关键发现是"先降后升"模式：跨域性能会先下降再恢复并提升，过早停止训练会低估泛化能力。此外，数据质量和结构都很重要——低质量解答普遍损害泛化，而经过验证的长 CoT 数据效果最好。这些发现对推理模型的训练策略选择有直接指导意义。

🔗 [原文链接](https://arxiv.org/abs/2604.06628) · 来源：arXiv AI · 关键词：#SFT泛化 #ChainOfThought #推理训练 #强化学习对比

### TurboAgent：LLM 驱动的涡轮机械气动设计自主多智能体框架（TurboAgent: An LLM-Driven Autonomous Multi-Agent Framework for Turbomachinery Aerodynamic Design）

涡轮机械的气动设计是一个复杂的多阶段耦合过程，涉及几何生成、性能预测、优化和高保真物理验证。现有的智能设计方法通常只关注单个阶段或依赖松耦合管线，难以实现全自主的端到端设计。TurboAgent 以 LLM 为核心进行任务规划和协调，由专业化 Agent 分别处理生成设计、快速性能预测、多目标优化和物理验证。这是 LLM 多智能体框架在传统工程领域的一个扎实落地案例，展示了 AI Agent 在高度专业化工业场景中的应用潜力。

### 将幻觉信号通过弱监督蒸馏到 Transformer 表征中（Weakly Supervised Distillation of Hallucination Signals into Transformer Representations）

现有的 LLM 幻觉检测方法依赖推理时的外部验证——需要标准答案、检索系统或辅助判断模型。这篇论文提出一个反向思路：能否在训练阶段就把外部监督信号蒸馏到模型自身的表征中，让模型仅凭内部激活就能在推理时检测幻觉？他们设计了一个弱监督框架，结合子串匹配、句子嵌入相似度和 LLM-as-judge 三种互补的锚定信号来标注生成内容。基于此框架，从 SQuAD v2 构建了 15,000 样本的数据集（10,500 训练/开发集）。这种"把幻觉检测能力内化"的思路，如果能规模化验证，对降低幻觉检测的推理成本意义重大。

## 🔧 开源项目

### ibu-boost：一个用"绝对拒绝"替代"相对排序"来选择分裂点的 GBDT 库（ibu-boost: a GBDT library where splits are absolutely rejected, not just relatively ranked）

传统 GBDT 实现总是选择增益最高的分裂点，即使最优候选几乎没用也会强行分裂，`min_gain_to_split` 是常见的补救手段但需要逐数据集调参。ibu-boost 基于 Nakanishi 2026 年论文"Screening Is Enough"中的 screening transform 思想，用绝对阈值拒绝机制替代相对比较，将这一原本为 Transformer 设计的核心理念迁移到了 GBDT 的分裂选择上。这意味着树只在分裂确实有意义时才分裂，无需手动调节增益阈值。虽然是个人独立实现的小项目，但思路新颖，值得关注 GBDT 方向的同学试试。
