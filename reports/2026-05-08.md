## 📄 研究论文

### 自然语言自编码器：无监督生成 LLM 激活的自然语言解释（Natural Language Autoencoders Produce Unsupervised Explanations of LLM Activations）

Anthropic 旗下 Alignment Forum 发布了一种名为"自然语言自编码器"（NLA）的无监督可解释性方法。NLA 由两个 LLM 模块组成：激活语言化器（AV）将模型内部激活映射为文本描述，激活重建器（AR）再将描述映射回激活向量，二者通过强化学习联合训练以重建残差流激活。尽管优化目标是激活重建，但生成的解释读起来像是对模型内部机制的合理解读，且随训练推进信息量持续增长。实际应用方面，NLA 已被用于 Claude Opus 4.6 的部署前安全审计，帮助诊断安全相关行为。这项工作为大模型的机械可解释性提供了一条可扩展的无监督路径，对 alignment 研究意义重大。

### 无损上下文管理：LCM 架构（LCM: Lossless Context Management）

研究者提出无损上下文管理（LCM），一种确定性的 LLM 记忆架构。基于 Opus 4.6 构建的编码智能体 Volt，在 OOLONG 长上下文评测中全面超越 Claude Code，覆盖 32K 到 1M token 的所有上下文长度区间。LCM 可视为递归语言模型（RLM）范式的验证与扩展，将符号递归分解为两种确定性的、引擎管理的机制。结果表明，递归上下文操作不仅能超越传统 LLM，还能击败拥有原生文件系统访问权限的前沿编码智能体。这对长上下文场景下的智能体架构设计具有重要参考价值。

### Agent Island：基于多智能体博弈的抗饱和抗污染基准（Agent Island: A Saturation- and Contamination-Resistant Benchmark from Multiagent Games）

静态能力基准面临饱和与数据污染问题，难以持续追踪模型进步。Agent Island 是一个多人模拟环境，语言模型智能体在其中进行合作、冲突与说服的博弈。该基准天然抗饱和——新模型总能尝试超越当前最强玩家；也抗污染——对手是自适应智能体而非固定题集。研究者使用贝叶斯 Plackett-Luce 模型对玩家排名并量化不确定性。在涉及 49 个模型的 999 场游戏中，openai/gpt-5.5 表现最为突出。这为 LLM 能力评估提供了一种动态、可持续的新范式。

### 面向 ARC-AGI-3 的可执行世界模型与编码智能体（Executable World Models for ARC-AGI-3 in the Era of Coding Agents）

研究者评估了一个面向 ARC-AGI-3 的编码智能体系统，该智能体维护一个可执行的 Python 世界模型，通过与历史观测验证、向更简洁抽象重构（作为 MDL 简洁性偏好的实用代理），并基于模型进行规划后再行动。系统设计刻意保持直接：使用脚本化控制器、预定义世界模型接口、验证程序和计划执行器，但不包含手工编码的游戏特定逻辑。在 25 个公开 ARC-AGI-3 游戏上报告了结果，每次游戏使用全新智能体实例。这项工作探索了程序合成与世界模型结合的 AGI 路径。

### AgentTrust：AI 智能体工具调用的运行时安全评估与拦截（AgentTrust: Runtime Safety Evaluation and Interception for AI Agent Tool Use）

现代 AI 智能体通过工具调用执行真实世界副作用——文件操作、shell 命令、HTTP 请求、数据库查询，一次不安全操作就可能造成不可逆损害。AgentTrust 是一个运行时安全层，在工具调用执行前进行拦截，返回结构化裁决：允许、警告、阻止或人工审查。它结合了 shell 反混淆规范化器和 SafeFix 建议机制，弥补了事后基准测试、静态护栏和基础设施沙箱各自的不足。这是将安全从"事后评估"推向"实时防护"的重要一步，对智能体的生产部署有直接实用价值。

### DecodingTrust-Agent 平台：AI 智能体的可控交互式红队测试平台（DecodingTrust-Agent Platform (DTap): A Controllable and Interactive Red-Teaming Platform for AI Agents）

随着 AI 智能体在高风险场景中的部署增多，真实世界已出现对手操纵智能体泄露 API 密钥、删除用户数据或发起未授权交易的案例。DTap 提供了一个可控、可复现的大规模风险评估环境，支持动态、不可信环境中的安全测试，涵盖外部工具、异构数据源和频繁用户交互场景。该平台填补了现有智能体安全评估在真实性和可控性之间的空白。对于需要在部署前系统性评估智能体安全风险的团队来说，这是一个值得关注的基础设施。

### LongSeeker：长程搜索智能体的弹性上下文编排（LongSeeker: Elastic Context Orchestration for Long-Horizon Search Agents）

长程搜索智能体在推理、调用工具和观察信息的过程中，工作上下文会快速膨胀。研究者提出 Context-ReAct 范式，将推理、上下文管理和工具使用统一在一个循环中，提供五种原子操作：Skip、Compress、Rollback、Snippet 和 Delete，让智能体根据当前任务相关性动态调整轨迹各部分的详细程度。这种自适应上下文管理思路直接回应了"上下文越多越好"这一朴素假设的局限性，对降低成本和减少错误都有实际意义。

### Strat-Reasoner：通过强化学习增强 LLM 在多智能体博弈中的策略推理（Strat-Reasoner: Reinforcing Strategic Reasoning of LLMs in Multi-Agent Games）

LLM 在多智能体博弈中表现不佳，因为最终结果取决于所有智能体的联合策略，其他智能体的非平稳性给推理评估和多步信用分配带来巨大挑战。Strat-Reasoner 引入递归推理范式，在推理过程中纳入对其他智能体的建模，通过强化学习框架提升 LLM 的策略推理能力。这弥补了现有单智能体 RL 方法及其多智能体扩展在博弈推理上的不足，为 LLM 在竞争性环境中的应用开辟了新方向。

🔗 [原文链接](https://arxiv.org/abs/2605.04906) · 来源：arXiv AI · 关键词：#策略推理 #多智能体 #强化学习 #博弈论

### Uno-Orchestra：基于选择性委派的精简智能体路由（Uno-Orchestra: Parsimonious Agent Routing via Selective Delegation）

现有 LLM 多智能体系统依赖刚性编排，要么是扁平的逐查询路由，要么是手工设计的任务分解。Uno-Orchestra 提出统一编排策略，选择性地分解任务并将子任务分派给合适的（模型, 原语）对，两个决策从真实工作者交互的 RL 轨迹中联合学习。在覆盖数学、代码、知识、长上下文和工具使用的 13 个基准上，Uno-Orchestra 达到 77.0% 宏平均 pass@1，比最强工作流基线高约 16%。这表明联合优化分解深度、工作者选择和推理预算是可行且有效的。

### 当上下文有害时：知识迁移对多智能体设计探索的交叉效应（When Context Hurts: The Crossover Effect of Knowledge Transfer on Multi-Agent Design Exploration）

"更多上下文总是更好"是智能体编排中的主流假设，但这项研究在 10 个任务、7 种上下文注入条件、超过 2700 次运行中发现了交叉效应：同一类型的制品在某些任务上提升设计探索（最高 20 倍权衡覆盖），在另一些任务上则严重降低探索（最高 46% 下降）。在部分任务中，一份无关文档的表现与所有相关制品一样好甚至更好。方向可由单一可测变量——无上下文时的基线探索程度——预测（Pearson r = -0.82, p < 0.001）。这对智能体系统的上下文策略设计提供了重要的实证警示。

🔗 [原文链接](https://arxiv.org/abs/2605.04361) · 来源：arXiv AI · 关键词：#多智能体 #上下文注入 #软件设计 #LLM编排

### AuditRepairBench：评估智能体修复中评估器通道排名不稳定性的配对执行追踪语料库（AuditRepairBench: A Paired-Execution Trace Corpus for Evaluator-Channel Ranking Instability in Agent Repair）

智能体修复排行榜在评估器重新配置后会发生重排，且部分重排由在内部候选修复选择中参考评估器信号的方法产生。研究者发布了 AuditRepairBench，包含 576,000 个注册单元（96,000 个已执行）的配对执行追踪语料库，通过四种可互换的筛选实现来操作化评估器通道阻断排名不稳定性。这项工作揭示了当前智能体评估体系中一个容易被忽视的系统性偏差，对排行榜的可信度提出了质疑。

🔗 [原文链接](https://arxiv.org/abs/2605.04624) · 来源：arXiv AI · 关键词：#智能体修复 #基准测试 #评估不稳定性 #排行榜

### 基于时间扩展交互图的手术团队动态实时建模（Actionable Real-Time Modeling of Surgical Team Dynamics via Time-Expanded Interaction Graphs）

手术团队表现源于技术执行与非技术技能（沟通、协调）的复杂交互，但当前手术 AI 主要建模视觉工作流信号。研究者提出使用时间扩展交互图建模手术团队动态，团队成员为时间索引节点，沟通交换定义有向边，时空扩展支持动态交互建模的同时允许静态图神经网络高效推理。模型预测程序效率偏差，为术中团队协作提供可操作的实时反馈。这是手术 AI 从纯视觉走向多模态团队理解的有意义尝试。

### SensingAgents：面向鲁棒 IMU 活动识别的多智能体协作框架（SensingAgents: A Multi-Agent Collaborative Framework for Robust IMU Activity Recognition）

基于 IMU 传感器的人体活动识别（HAR）是移动健康和智能环境的基石，但深度学习方法面临标注依赖、位置歧义和缺乏透明推理等问题。SensingAgents 将 LLM 驱动的智能体组织为专门角色：针对不同佩戴位置（手臂、手腕、腰带、口袋）的分析师智能体组，以及解决传感器冲突的辩护智能体对。这种多智能体协作方式为传感器数据理解引入了可解释的推理链，展示了 LLM 智能体在传统信号处理领域的新应用可能。
