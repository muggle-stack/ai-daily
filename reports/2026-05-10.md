### LLM Agent 自动发现交换关联密度泛函（Agentic Discovery of Exchange-Correlation Density Functionals）

密度泛函理论（DFT）中交换关联泛函的设计一直依赖人类研究者的物理直觉和经验拟合。这篇论文提出了一个 Agent 搜索系统，让 LLM 在进化历史引导下提出结构化的泛函形式变更，通过"计划-执行-总结"的迭代循环来优化泛函性能。系统以标准热化学数据集为优化目标，实现了可度量的改进。这是 LLM 作为科学发现工具的又一个有力案例——不是替代物理学家，而是系统化地探索人类难以穷举的设计空间。

### 比特校准测试时缩放：量化推理模型的自适应计算分配（BitCal-TTS: Bit-Calibrated Test-Time Scaling for Quantized Reasoning Models）

4-bit 量化让大型推理模型在有限内存和延迟预算下变得实用，但量化会扭曲驱动自适应测试时计算分配的在线信号。具体表现为：模型可能在推理尚未稳定时就过早停止，输出看似合理但实际错误的答案。BitCal-TTS 提出了一个轻量运行时控制器，结合 token 级不确定性代理、推理轨迹稳定性检测和比特条件置信度重校准，在固定 token 预算下显著改善了量化模型的推理质量。这项工作填补了"量化部署"与"测试时计算"两个热门方向之间的空白。

### 信念记忆：部分可观测下的 Agent 记忆（Belief Memory: Agent Memory Under Partial Observability）

现有 LLM Agent 的外部记忆通常将每次观察存储为单一确定性结论（比如从临时错误推断"API X 失败了"），但现实观察本质上是部分的、模糊的。这种做法会引发自我强化错误：Agent 基于存储的结论行动，从不重新审视替代解释。BeliefMem 将记忆范式从"每次观察提交一个结论"转变为"保留多个候选结论及其概率"，让 Agent 在不确定性下做出更稳健的决策。这是将贝叶斯思维引入 Agent 记忆系统的一次有意义的尝试。

### BALAR：贝叶斯主动推理的 Agent 循环（BALAR: A Bayesian Agentic Loop for Active Reasoning）

当前大多数 LLM 系统在多轮对话中是被动反应式的，缺乏对"缺少什么信息"和"下一步该问什么"的原则性推理。BALAR 提出了一个任务无关的外循环算法，无需微调即可实现 LLM Agent 与用户之间的结构化多轮交互。它维护对潜在状态的结构化信念，通过最大化期望互信息来选择澄清问题，并在当前状态表示不足时动态扩展。这为构建真正"会提问"的 AI 助手提供了理论基础。

### PRISM：感知推理交织的序列决策框架（PRISM: Perception Reasoning Interleaved for Sequential Decision Making）

将 LLM 具身 Agent 从纯文本环境扩展到复杂多模态场景仍是一大挑战，核心问题在于视觉语言模型（VLM）常常忽略任务关键信息。PRISM 通过动态问答管道将感知（VLM）和决策（LLM）紧密耦合：LLM 不是被动接受 VLM 的描述，而是主动批评、用目标导向的问题探测 VLM，并合成紧凑的场景描述。在 ALFWorld 和 Room-to-Room 基准上取得了显著提升。这种闭环交互模式比单次"看一眼就决策"更接近人类的认知过程。

🔗 [原文链接](https://arxiv.org/abs/2605.05407) · 来源：arXiv AI · 关键词：#具身智能 #多模态 #VLM #闭环感知

### 从历史到状态：LLM Agent 的常量上下文技能学习（From History to State: Constant-Context Skill Learning for LLM Agents）

LLM Agent 面临隐私-成本-能力的三角矛盾：云端模型执行多步工作流效果好但暴露敏感中间上下文，本地模型保护隐私但可靠性不足。本文提出常量上下文技能学习框架，将可复用的工作流程序学习到轻量任务族模块中，推理时仅依赖当前观察和紧凑状态块，用确定性追踪器替代不断增长的历史记录。这种"上下文到权重"的思路既降低了推理成本，又为隐私保护提供了新路径。

### AgenticRAG：面向企业知识库的 Agent 检索（AgenticRAG: Agentic Retrieval for Enterprise Knowledge Bases）

标准 RAG 管道将过多的接地负担放在搜索栈上，限制了语言模型只能使用检索过程深处选定的固定候选集。AgenticRAG 在现有企业搜索基础设施之上叠加轻量 Agent 框架，为推理 LLM 配备搜索、查找、打开和摘要工具，使模型能自主迭代检索、文档内导航和证据分析。在三个开放基准上取得显著提升：BRIGHT 上 recall@1 达 49.6%（比最佳嵌入基线高 21.8 个百分点），事实性评分 0.96。对企业级 RAG 落地有直接参考价值。

### 检索条件拓扑选择与多 Agent 代码生成的预算守恒（Retrieval-Conditioned Topology Selection with Provable Budget Conservation for Multi-Agent Code Generation）

多 Agent LLM 代码生成系统面临一个基本路由问题：最优编排拓扑取决于代码的结构复杂度，但现有系统在选择拓扑时不参考代码库。RGAO 架构通过从层次化代码索引中提取结构复杂度向量，在选择编排拓扑前闭合这个循环。系统在 Code-Agent 多 Agent 框架中运行，子 Agent 受六维预算向量的形式化合约约束。这是将复杂度条件路由与形式化预算保证两条研究线结合的首次尝试。

### 部分证据基准：Agent 系统中授权受限证据的评测（Partial Evidence Bench: Benchmarking Authorization-Limited Evidence in Agentic Systems）

企业 Agent 越来越多地在有权限约束的检索系统中运行，访问控制可能被正确执行，但系统仍然产出看似完整实则遗漏了授权边界外关键证据的答案。Partial Evidence Bench 提供了 72 个任务，覆盖尽职调查、合规审计和安全事件响应三个场景族，配备 ACL 分区语料库和完整性判断预言机。这个基准填补了一个重要空白：评估 Agent 系统在"能看到什么"受限时是否诚实地报告信息缺口。

### 多 Agent AI 系统中的授权传播：身份治理即基础设施（Authorization Propagation in Multi-Agent AI Systems: Identity Governance as Infrastructure）

Agent AI 的安全讨论过度聚焦于提示注入，但多 Agent 系统还存在一个独立的授权问题：当非人类主体跨边界检索数据、委托任务和综合结果时，如何维护授权不变量。论文将此形式化为"授权传播"问题，识别出传递委托、聚合推断和时间有效性三个子问题，并推导出七项结构性要求。这不是 RBAC/ABAC 能解决的——当 Agent 链式调用时，权限的语义会发生质变。

### AlphaCrafter：面向截面量化交易的全栈多 Agent 框架（AlphaCrafter: A Full-Stack Multi-Agent Framework for Cross-Sectional Quantitative Trading）

金融市场的非平稳性要求因子发现、体制自适应选择和风险约束执行的持续耦合，但现有方法通常在静态或孤立假设下优化这些组件。AlphaCrafter 提出全栈多 Agent 框架，将因子挖掘视为持续过程而非一次性搜索，避免了角色扮演式 Agent 架构引入的行为噪声。这是 LLM 多 Agent 系统在量化金融领域的一次系统性应用尝试，强调的是工程严谨性而非拟人化模拟。

### FinAgent-RAG：面向金融文档问答的 Agent 检索增强生成（Agentic Retrieval-Augmented Generation for Financial Document Question Answering）

金融文档问答需要对异构证据（结构化表格、文本叙述、脚注）进行复杂的多步数值推理，传统单次"检索-生成"范式难以应对。FinAgent-RAG 编排迭代检索-推理循环并加入自验证机制，集成了三项领域特定创新：对比金融检索器（使用困难负样本训练）、专门的数值推理模块和自我验证环节。这种为金融精度需求专门工程化的 Agent RAG 框架，比通用方案更贴合实际业务场景。

## 💡 行业观点

### 谁来为认知劳动定价？关于计算锚定工资的立场（Who Prices Cognitive Labor in the Age of Agents? A Position on Compute-Anchored Wages）

一个直觉性的观点是：AI Agent 可以近零边际成本复制，因此构成无限弹性供给的劳动投入，会将认知劳动工资压至零。本文认为这个框架在机制上是错误的：Agent 不是劳动，而是将计算资本转化为有效认知劳动单位的生产技术。一旦认识到这一点，锚定均衡工资的弹性供给边际就从劳动市场迁移到了计算资本市场。论文基于教科书要素定价框架推导了正式模型，对理论和政策都有启发意义。这是一个值得认真对待的经济学视角——AI 对劳动市场的影响可能不是"替代"那么简单。
