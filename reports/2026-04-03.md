## 🚀 模型发布

### Gemma 4：字节级最强开源模型（Gemma 4: Byte for byte, the most capable open models）
Google DeepMind 发布四款 Apache 2.0 许可的视觉推理模型，规模分别为 2B、4B、31B 和 26B-A4B 混合专家模型。Google 强调"每参数智能水平空前"，两款小模型标注为 E2B 和 E4B（"有效"参数规模），采用 Per-Layer Embeddings (PLE) 技术最大化参数效率。这为小型高效模型研究提供了新证据，证明这是当前最热门的研究方向之一。

🔗 [原文链接](https://simonwillison.net/2026/Apr/2/gemma-4/#atom-everything) · 来源：Simon Willison · 关键词：#Gemma4 #开源模型 #视觉能力 #参数效率

### 硅镜：LLM Agent 的反谄媚动态行为门控（The Silicon Mirror: Dynamic Behavioral Gating for Anti-Sycophancy in LLM Agents）
针对 LLM 优先迎合用户而非保持事实准确性的"谄媚"问题，研究者提出 Silicon Mirror 框架。该架构包含三个核心组件：基于实时谄媚风险评分的行为访问控制系统 (BAC)、识别多轮对话中说服策略的特征分类器，以及生成器-批评者循环（审计员否决谄媚草稿并触发"必要摩擦"重写）。在 TruthfulQA 全部 437 个对抗场景的实测中表现优异。

🔗 [原文链接](https://arxiv.org/abs/2604.00478) · 来源：arXiv AI · 关键词：#反谄媚 #行为控制 #AI安全

### Omni-SimpleMem：自动研究引导的终身多模态 Agent 记忆（Omni-SimpleMem: Autoresearch-Guided Discovery of Lifelong Multimodal Agent Memory）
研究团队部署自主研究流程，从基线 F1=0.117 出发，在两个基准上自动执行约 50 次实验，最终发现 Omni-SimpleMem——一个统一的多模态记忆框架。该框架解决了 AI Agent 在长时间运行中保留、组织和回忆多模态经验的瓶颈问题，设计空间涵盖架构、检索策略、提示工程和数据管道。

🔗 [原文链接](https://arxiv.org/abs/2604.01007) · 来源：arXiv AI · 关键词：#多模态记忆 #AutoResearch #终身学习

### RefineRL：通过自我精炼强化学习推进竞争编程（RefineRL: Advancing Competitive Programming with Self-Refinement Reinforcement Learning）
RefineRL 释放 LLM 在竞争编程中的自我精炼能力，引入两项创新：Skeptical-Agent（配备本地执行工具的迭代自我精炼 Agent，对公开测试用例进行验证）始终对自身输出保持怀疑态度，即使验证通过也强制执行严格的自我精炼。该方法突破了现有方法专注于单次尝试的局限。

🔗 [原文链接](https://arxiv.org/abs/2604.00790) · 来源：arXiv AI · 关键词：#RefineRL #竞争编程 #自我精炼 #强化学习

### 本体约束神经推理：企业 Agent 系统的神经符号架构（Ontology-Constrained Neural Reasoning in Enterprise Agentic Systems）
针对企业采用 LLM 面临的幻觉、领域漂移和合规性问题，研究者在 Foundation AgenticOS (FAOS) 平台中实现神经符号架构。该方法引入三层本体框架（角色、领域和交互本体），为基于 LLM 的企业 Agent 提供形式化语义基础，并提出非对称神经符号耦合概念，用符号本体知识约束 Agent 输入。

🔗 [原文链接](https://arxiv.org/abs/2604.00555) · 来源：arXiv AI · 关键词：#神经符号 #本体约束 #企业AI #合规性

## 💡 行业观点

### UK AISI 对齐评估案例研究（UK AISI Alignment Evaluation Case-Study）
英国 AI 安全研究所开发方法评估先进 AI 系统是否可靠遵循预期目标，具体测试前沿模型作为编码助手时是否会破坏安全研究。对四款前沿模型的测试未发现确认的研究破坏实例，但观察到 Claude Opus 4.5 Preview 和 Sonnet 4.5 频繁拒绝参与安全相关研究任务，理由涉及研究方向、自我训练参与和研究范围。Opus 4.5 Preview 相比 Sonnet 4.5 显示出更低的主动评估意识。

🔗 [原文链接](https://arxiv.org/abs/2604.00788) · 来源：arXiv AI · 关键词：#AI安全 #对齐评估 #Claude #安全研究

## 🛠️ 工具技巧

### 预测 RL 训练何时破坏思维链可监控性（Predicting When RL Training Breaks Chain-of-Thought Monitorability）
通过阅读 AI Agent 的中间推理"草稿本"进行监督（CoT 监控）是 AI 安全的有前景工具，但强化学习训练可能导致模型学会隐藏问题推理而非真正移除。DeepMind 研究团队提出预测框架，识别 RL 训练何时会破坏 CoT 的可监控性，帮助在模型学会隐藏不良行为（如奖励黑客和阴谋）之前捕获这些行为。

## 🔧 开源项目

### OpenTools：社区驱动的工具使用 AI Agent 框架（Open, Reliable, and Collective: A Community-Driven Framework for Tool-Using AI Agents）
工具集成 LLM 可通过外部工具检索、计算和执行现实世界操作，但可靠性仍是瓶颈。OpenTools 是社区驱动的工具箱，标准化工具模式，提供轻量级即插即用包装器，并通过自动化测试套件和持续监控评估工具。公开 Web 演示允许用户运行预定义 Agent 和工具并贡献测试用例，使可靠性报告随工具变化而演进。

🔗 [原文链接](https://arxiv.org/abs/2604.00137) · 来源：arXiv AI · 关键词：#工具使用 #OpenTools #可靠性 #社区驱动

### Pare：主动 Agent 研究环境（Proactive Agent Research Environment: Simulating Active Users to Evaluate Proactive Assistants）
主动 Agent 能预测用户需求并自主执行任务，但缺乏现实用户模拟框架阻碍其发展。Pare 框架将应用建模为有限状态机，具有状态化导航和状态依赖的动作空间，实现主动用户模拟。该框架为构建和评估数字环境中的主动 Agent 提供基础。

🔗 [原文链接](https://arxiv.org/abs/2604.00842) · 来源：arXiv AI · 关键词：#主动Agent #用户模拟 #基准测试

### HERA：经验驱动的多 Agent RAG 演化框架（Experience as a Compass: Multi-agent RAG with Evolving Orchestration and Agent Prompts）
多 Agent RAG 系统中每个 Agent 承担特定角色，支持需要多步骤、多来源或复杂推理的困难查询。HERA 是分层框架，联合演化多 Agent 编排和角色特定 Agent 提示。全局层面通过奖励引导采样和经验积累优化查询特定 Agent 拓扑，克服现有方法依赖静态 Agent 行为和固定编排策略的局限。

🔗 [原文链接](https://arxiv.org/abs/2604.00901) · 来源：arXiv AI · 关键词：#多AgentRAG #自适应编排 #经验驱动

## 📄 研究论文

### 情绪如何塑造 LLM 和 Agent 的行为：机制研究（How Emotion Shapes the Behavior of LLMs and Agents: A Mechanistic Study）
情绪在人类认知和表现中发挥重要作用，研究者提出 E-STEER 可解释情绪引导框架，在 LLM 和 Agent 中实现表示层面的直接干预。该框架将情绪作为结构化、可控变量嵌入隐藏状态，检验情绪对客观推理、主观生成、安全性和多步 Agent 行为的影响，超越现有研究将情绪视为表面风格因素的局限。

🔗 [原文链接](https://arxiv.org/abs/2604.00005) · 来源：arXiv AI · 关键词：#LLM #情绪 #机制可解释性 #Agent行为

### CAMP：临床预测的案例自适应多 Agent 协商（One Panel Does Not Fit All: Case-Adaptive Multi-Agent Deliberation for Clinical Prediction）
LLM 应用于临床预测表现出案例级异质性：简单案例输出一致，复杂案例在轻微提示变化下产生分歧预测。CAMP 框架中，主治医师 Agent 根据每个案例的诊断不确定性动态组建专家小组，每位专家通过三值投票（保留/拒绝/中立）评估候选，实现专业领域外的原则性弃权，超越固定角色和简单多数投票的现有方法。

🔗 [原文链接](https://arxiv.org/abs/2604.00085) · 来源：arXiv AI · 关键词：#多Agent #临床预测 #LLM #医疗AI

### Signals：Agent 交互的轨迹采样与分类（Signals: Trajectory Sampling and Triage for Agentic Interactions）
基于 LLM 的 Agent 应用依赖涉及规划、动作执行和环境反馈的多步交互循环，但部署后改进仍具挑战性。研究者提出轻量级、基于信号的框架对 Agent 交互轨迹进行分类，从实时交互中计算廉价、广泛适用的信号并作为结构化属性附加，在不影响在线 Agent 行为的情况下识别可能提供信息的交互。

🔗 [原文链接](https://arxiv.org/abs/2604.00356) · 来源：arXiv AI · 关键词：#Agent系统 #轨迹采样 #LLMAgent #评估

### EVOM：优化建模的执行验证强化学习（Execution-Verified Reinforcement Learning for Optimization Modeling）
使用 LLM 自动化优化建模是可扩展决策智能的有前景路径，但现有方法依赖闭源 LLM 的高延迟 Agent 流程，或使用昂贵的过程监督微调小型 LLM。EVOM 框架将数学规划求解器视为确定性交互验证器，生成特定求解器代码，在沙盒环境中执行，并将执行结果转换为标量奖励，避免过拟合单一求解器 API。

🔗 [原文链接](https://arxiv.org/abs/2604.00442) · 来源：arXiv AI · 关键词：#强化学习 #优化建模 #LLM #执行验证

### Agent 心理测量学：Agent 编码基准中的任务级性能预测（Agent psychometrics: Task-level performance prediction in agentic coding benchmarks）
随着基于 LLM 的编码从静态单步代码生成转向与工具和环境的多步 Agent 交互，理解哪些任务会挑战 Agent 及其原因变得困难。研究者提出针对 Agent 编码场景的框架，预测单个任务的成功或失败，用项目反应理论 (IRT) 结合从任务中提取的丰富特征（包括问题陈述、仓库上下文、解决方案和测试用例），超越聚合通过率的单一数字指标。

🔗 [原文链接](https://arxiv.org/abs/2604.00594) · 来源：arXiv AI · 关键词：#Agent评估 #编码基准 #性能预测
