## 📄 研究论文

### 超越智能体边界的"制品即记忆"（Artifacts as Memory Beyond the Agent Boundary）

这篇论文从具身认知的角度出发，提出了一个有趣的观点：智能体的记忆不一定全靠"脑子"，环境本身也能充当记忆。作者在强化学习框架下对此进行了数学形式化，定义了一类称为"artifact"的观测信号，并证明这些信号可以减少智能体表征历史所需的信息量。实验表明，当智能体能观察到空间路径时，学习高效策略所需的内存显著降低——而且这种效果是通过感知流隐式产生的，并非刻意设计。这项工作为 Agent 的记忆机制设计提供了新的理论基础，暗示我们或许不需要把所有东西都塞进上下文窗口，善用环境信息就够了。

### 自进化智能体的首个评测基准 SEA-Eval（SEA-Eval: A Benchmark for Evaluating Self-Evolving Agents Beyond Episodic Assessment）

当前 LLM Agent 的一个核心痛点是"每次任务都从零开始"——没有经验积累，没有跨任务的策略优化。这篇论文正式定义了"自进化智能体"（SEA）的概念，并推出了首个配套评测基准 SEA-Eval。该基准从两个维度评估 Agent：单任务执行的可靠性和长期演化性能，通过将任务组织为顺序流并追踪成功率与 Token 消耗的变化来量化"进化增益"。这填补了一个重要的评测空白——我们终于有了衡量 Agent 是否真的在"成长"的标尺。对于做 Agent 系统的团队来说，这个基准值得纳入评估体系。

🔗 [原文链接](https://arxiv.org/abs/2604.08988) · 来源：arXiv AI · 关键词：#自进化Agent #SEA-Eval #LLMAgent #跨任务演化

### 编程 Agent 知道何时该求助吗？HiL-Bench 来检验（HiL-Bench (Human-in-Loop Benchmark): Do Agents Know When to Ask for Help?）

这篇论文直击当前编程 Agent 的一个被忽视的致命缺陷：它们在信息不完整或需求模糊时不会主动求助，而是"猜着做"。现有基准给的都是清晰无歧义的指令，只看执行结果对不对，所以一个靠蒙对的 Agent 和一个真正理解需求的 Agent 得分一样。HiL-Bench 专门设计了包含人工验证的"阻断点"——缺失信息、模糊需求、矛盾指令——来测试 Agent 的"选择性升级"能力，即知道什么时候该自己干、什么时候该问人。这对实际工程场景极其重要，因为在真实开发中，需求从来不会完美无缺。可以说，这个基准衡量的不是能力，而是判断力。

### 优势引导扩散：为基于模型的强化学习注入远见（Advantage-Guided Diffusion for Model-Based Reinforcement Learning）

基于自回归世界模型的 MBRL 长期受累积误差困扰，扩散世界模型通过联合生成轨迹片段缓解了这一问题，但现有的扩散引导方法要么只用策略信息丢掉了价值信号，要么基于奖励但在短视野下表现短视。AGD-MBRL 提出用智能体的优势估计来引导反向扩散过程，让采样集中在预期长期回报更高的轨迹上，即使生成窗口之外的收益也能被考虑进来。论文提出了两种引导方式：Sigmoid 优势引导（SAG）和指数优势引导（EAG），并给出了理论证明。这为扩散模型在决策领域的应用提供了更优雅的价值整合方案。

### E3-TIR：让 LLM 更高效地学会使用工具推理（E3-TIR: Enhanced Experience Exploitation for Tool-Integrated Reasoning）

LLM 的工具集成推理（TIR）训练面临两难：Zero-RL 探索效率低且容易模式退化，SFT-then-RL 数据成本高且因低熵坍缩导致能力天花板。E3-TIR 提出了一种"热身范式"，将训练动态整合为三类经验：专家前缀、专家引导和自主探索。核心思路是围绕专家"锚点"进行多样化分支探索，并通过混合策略平衡模仿与创新。这种方法巧妙地在"跟着学"和"自己闯"之间找到了平衡点，有望成为 Agent 训练早期阶段的通用方案。

🔗 [原文链接](https://arxiv.org/abs/2604.09455) · 来源：arXiv AI · 关键词：#LLM #工具集成推理 #E3-TIR #训练范式

### 过程奖励智能体：为知识密集推理提供实时逐步引导（Process Reward Agents for Steering Knowledge-Intensive Reasoning）

在知识密集型领域做推理，中间步骤往往无法像数学或代码那样本地验证——你可能需要综合大量外部知识才能判断某一步对不对。现有的过程奖励模型（PRM）都是事后打分，无法融入动态推理过程。PRA（Process Reward Agents）改变了这一点：它在测试时为冻结的策略模型提供基于领域知识的、在线的、逐步的奖励信号。相比之前的检索增强 PRM，PRA 是主动参与推理过程而非被动评估，这对减少知识密集推理中的错误传播意义重大。

🔗 [原文链接](https://arxiv.org/abs/2604.09482) · 来源：arXiv AI · 关键词：#过程奖励模型 #知识密集推理 #PRA #检索增强

### 当 AI 都用同一个算法：LLM 在协调博弈中的"算法单一文化"（Strategic Algorithmic Monoculture: Experimental Evidence from Coordination Games）

当越来越多的 AI Agent 在多智能体环境中协作时，一个有趣的问题浮出水面：它们会不会因为"想法太像"而出问题？这篇论文区分了"初级算法单一文化"（基线行为相似性）和"策略性算法单一文化"（根据激励调整相似性），并在人类和 LLM 上都做了实验。结果发现 LLM 天然具有很高的基线相似性，在需要协调一致时表现极好，但在需要差异化时明显不如人类。这对多 Agent 系统设计是一个重要警示：当所有 Agent 都基于相似的 LLM 时，系统可能在需要多样性的场景中表现脆弱。

### OpenKedge：用执行合约约束 AI Agent 的状态变更（OpenKedge: Governing Agentic Mutation with Execution-Bound Safety and Evidence Chains）

自主 AI Agent 直接调用 API 执行状态变更，缺乏足够的上下文、协调和安全保障——这是当前 Agent 架构的根本性缺陷。OpenKedge 协议将"变更"重新定义为一个受治理的过程：Agent 必须先提交声明式意图提案，系统根据确定性状态、时间信号和策略约束进行评估，通过后编译为严格限定操作范围、资源和时间的"执行合约"。这把安全从"事后补救"变成了"事前约束"，对于在生产环境中部署 Agent 的团队来说，这种架构思路非常值得借鉴。

🔗 [原文链接](https://arxiv.org/abs/2604.08601) · 来源：arXiv AI · 关键词：#AIAgent #安全治理 #OpenKedge #mutation协议

### PilotBench：LLM 能在安全约束下预测飞行轨迹吗？（PilotBench: A Benchmark for General Aviation Agents with Safety Constraints）

当 LLM 走向具身 AI，一个根本问题是：基于文本训练的模型能否可靠地推理复杂物理过程并遵守安全约束？PilotBench 用 708 条真实通用航空轨迹、覆盖 9 个飞行阶段、同步 34 通道遥测数据来回答这个问题。基准引入了 Pilot-Score 复合指标，其中 60% 权重给了安全相关的预测准确性。这是一个极具挑战性的测试场景，直接检验 LLM 在安全关键领域的物理推理能力边界。

### DRBENCHER：你的 Agent 能识别实体、检索属性并完成计算吗？（DRBENCHER: Can Your Agent Identify the Entity, Retrieve Its Properties and Do the Math?）

深度研究 Agent 越来越多地需要交替进行网页浏览和多步计算，但现有基准把这两种能力分开评估，留下了一个评测盲区。DRBENCHER 是一个合成基准生成器，专门生成同时需要浏览和计算的问题，并通过四重标准保证质量：可验证性（金标答案由参数化代码在知识图谱值上执行得出）、复杂性（多跳实体识别+属性检索+领域计算）、难度（两阶段级联过滤）和多样性（贪心最大最小嵌入过滤）。这为评估"真正能做研究"的 Agent 提供了更贴近现实的测试工具。

🔗 [原文链接](https://arxiv.org/abs/2604.09251) · 来源：arXiv AI · 关键词：#深度研究Agent #DRBENCHER #网页浏览 #计算推理

### 约束感知纠正记忆：让药物发现 Agent 不再"顾此失彼"（Constraint-Aware Corrective Memory for Language-Based Drug Discovery Agents）

LLM 驱动的自主药物发现 Agent 面临一个根本性控制难题：Agent 逐步规划，但任务成败取决于最终候选集是否整体满足协议级要求（集合大小、多样性、结合质量、可开发性等）。现有系统依赖冗长的原始历史和模糊的自我反思，导致错误定位不精确、Agent 状态越来越嘈杂。CACM（约束感知纠正记忆）通过结构化的约束追踪和精确的失败定位来解决这个问题。这项工作展示了 Agent 记忆机制在垂直领域的精细化设计方向。

### Camera Artist：用多 Agent 框架生成具有电影语言的叙事视频（Camera Artist: A Multi-Agent Framework for Cinematic Language Storytelling Video Generation）

现有的多 Agent 影视制作系统虽然能从剧本生成视频，但往往缺乏对镜头间叙事连贯性和电影语言的显式建模，导致故事碎片化、影像质感不足。Camera Artist 在已有的 Agent 流水线基础上引入了专门的"电影摄影镜头 Agent"，通过递归分镜生成增强镜头间的叙事连续性，并注入电影语言元素来提升影像品质。这是 AI 视频生成从"能生成"走向"会讲故事"的一个有意义的尝试。

### 空间推理与行动之间的鸿沟：Spatial-Gym 逐步评估 Agent（Mind the Gap Between Spatial Reasoning and Acting! Step-by-Step Evaluation of Agents With Spatial-Gym）

空间推理是导航和机器人的核心能力，但现有基准都是一次性生成完整解，不像人类那样在交互环境中逐步决策。Spatial-Gym 将空间约束推理隔离为 2D 网格寻路的序列决策任务，支持回溯，并在三种设置下评估了 8 个模型。结果相当扎眼：表现最好的 GPT-OSS 120B 解题率仅 16.0%，比人类基线低了 82 个百分点。这清楚地表明，当前 LLM 在交互式空间推理上与人类差距巨大，500 个测试 episode 的数据让这个结论很有说服力。

🔗 [原文链接](https://arxiv.org/abs/2604.09338) · 来源：arXiv AI · 关键词：#空间推理 #Spatial-Gym #Agent #导航

### 超图神经网络加速最小不可满足子集枚举（Hypergraph Neural Networks Accelerate MUS Enumeration）

枚举最小不可满足子集（MUS）是约束满足问题中的基础任务，核心挑战在于搜索空间的指数级增长。现有机器学习方法依赖显式的变量-约束关系，限制了适用范围。这篇论文提出了一种领域无关的方法：用超图神经网络（HGNN）增量构建超图，以约束为顶点、已枚举的 MUS 为超边，通过强化学习训练 HGNN Agent 来加速枚举过程。方法的领域无关性是最大亮点，意味着它可以迁移到不同类型的约束满足问题中。
